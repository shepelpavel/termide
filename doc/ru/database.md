# Просмотр баз данных

В TermIDE встроен **просмотрщик баз данных только для чтения** для **SQLite**,
**PostgreSQL** и **MySQL/MariaDB**. Он подключается по нативному протоколу (без
внешних клиентских утилит), показывает список таблиц базы и выводит строки в
прокручиваемой таблице с сортировкой и фильтрацией по столбцам.

Просмотрщик никогда не пишет в базу: выполняются только запросы `SELECT` и
запросы к каталогу.

## Открытие подключения

Подключения — это обычные **закладки**, путь которых является URL базы данных.
Добавьте закладку (меню «Закладки» → «Добавить») и укажите URL подключения в
поле *Путь*:

| Движок | Пример URL |
|--------|------------|
| SQLite | `sqlite:///абсолютный/путь/app.db` |
| PostgreSQL | `postgres://user@host:5432/mydb` |
| MySQL / MariaDB | `mysql://user@host:3306/mydb` |

Активация закладки базы данных открывает панель «База данных» и подключается в
фоне — интерфейс при подключении не блокируется.

### Пароли

Как и для закладок SFTP/FTP, пароль не обрабатывается особым образом: закладка
хранит ровно тот URL, который вы ввели. Можно либо положиться на беспарольный
способ аутентификации (PostgreSQL `~/.pgpass`/peer/trust, Unix-сокет,
`PGPASSWORD`, …), либо включить пароль прямо в URL
(`postgres://user:secret@host/db`). Если вы его впишете, учтите, что закладка
сохраняется в `bookmarks.toml` как есть — обращайтесь с этим файлом
соответственно.

## Расположение

```
┌─ База данных ───────────────────────────────────────┐
│ Таблица ▾ users                                      │
│ id │ name  │ score │ active                          │
│  1 │ alice │ 1.5   │ true                             │
│  2 │ bob   │ 2.25  │ false                            │
│  3 │ carol │ NULL  │ true                             │
└──────────────────────────────────────────────────────┘
```

Сверху — **селектор таблицы**, остальное занимает **таблица данных**. У таблицы
двумерный курсор по ячейкам: подсвеченная ячейка — цель для сортировки,
фильтрации и копирования. В общем статус-баре показываются текущий диапазон,
полное число строк, активная сортировка и фильтр, например:
`app.db · users · rows 1–200 of 1203 · sort: name ↑ · filter: 1`.

## Клавиши

| Клавиша | Действие |
|---------|----------|
| `Tab` | Переключение фокуса между селектором таблицы и таблицей |
| `Enter` / `Space` (на селекторе) | Открыть выпадающий список таблиц |
| `↑ ↓ ← →` | Перемещение курсора ячейки (автопрокрутка строк и столбцов) |
| `PageUp` / `PageDown` | Перемещение на экран; на краях подгружает следующее/предыдущее окно |
| `Home` / `End` | Перейти к первой / последней строке |
| `S` | Сортировка по текущему столбцу — цикл: по возрастанию → по убыванию → без сортировки |
| `F` | Фильтр по текущему столбцу (открывает диалог фильтра) |
| `Ctrl+F` | Сбросить все фильтры |
| `Space` / `F12` (на таблице) | Показать строку целиком (ключ/значение) с вариантами копирования |
| `Y` | Скопировать значение текущей ячейки |
| `Ctrl+Y` | Скопировать текущую строку как значения через табуляцию |
| `F5` / `Ctrl+R` | Обновить список таблиц и перезагрузить текущий вид |

Клавиши действий настраиваются в секции `[database.keybindings]` конфига и
отображаются в панели хелпа. Клавиши навигации (Tab, стрелки, PageUp/Down,
Home/End) фиксированы.

**Мышь:** клик по селектору таблицы открывает его, клик по заголовку столбца
переключает сортировку, клик по ячейке перемещает курсор.

## Сортировка

Нажмите `s` на столбце для серверной сортировки по нему. Цикл: **по возрастанию
→ по убыванию → без сортировки**; «без сортировки» не добавляет `ORDER BY`, что
дешевле всего на больших таблицах. У отсортированного столбца в заголовке
рисуется стрелка `↑`/`↓`. Сортировка по одному столбцу за раз.

> В PostgreSQL листание *неотсортированной* таблицы может возвращать строки в
> слегка разном порядке между страницами (естественный порядок движка не
> стабилен между `OFFSET`). Примените сортировку, если нужен стабильный порядок.

## Фильтрация

Нажмите `f` на столбце, чтобы добавить условие. Доступные операторы зависят от
типа столбца:

- **Текст:** содержит, начинается с, заканчивается на, =, ≠, пусто, не пусто
- **Число / дата:** =, ≠, >, ≥, <, ≤, пусто, не пусто
- **Логический:** =, ≠, пусто, не пусто

`содержит`/`начинается с`/`заканчивается на` регистронезависимы. Условия по
разным столбцам объединяются через **AND**; повторная фильтрация того же
столбца заменяет его условие. Нажмите `F`, чтобы сбросить всё. Значения всегда
передаются как связанные параметры.

## Детали строки

Нажмите `Space` на строке, чтобы открыть диалог деталей со всеми столбцами в
виде ключ/значение — удобно для длинного текста, JSON или широких строк, не
помещающихся в таблицу. Из диалога строку можно скопировать как **TSV**,
**JSON** или оператор **INSERT**.

## Пагинация

Строки загружаются окнами; в памяти держится только текущее окно, поэтому даже
очень большие таблицы остаются отзывчивыми. При выходе за конец окна следующее
подгружается автоматически. Полное число строк (с учётом активного фильтра)
вычисляется в фоне и появляется в статус-баре, как только готово.

## Ограничения (на данный момент)

- Только чтение — без редактирования и произвольного SQL.
- Экзотические типы столбцов (JSON, массивы, UUID, временные метки, беззнаковые
  целые) в этой версии показываются как пустые/`NULL`; обычные скалярные типы
  выводятся полностью.
- Выбор схемы (PostgreSQL) по умолчанию использует текущую схему.
- Пока нет запроса пароля в приложении — используйте беспарольный способ
  аутентификации или включите пароль в URL (см. выше).
