# Текстовый редактор

Панель текстового редактора предоставляет функциональный редактор для работы с текстовыми файлами с поддержкой подсветки синтаксиса для различных языков программирования.

## Основные возможности

- **Подсветка синтаксиса**: Автоматическая подсветка для популярных языков программирования (Rust, Python, JavaScript, C/C++, Go и др.)
- **Визуализация Git Diff**: Визуализация изменений в реальном времени по сравнению с HEAD с фоновой подсветкой номеров строк (зелёный фон для добавленных, жёлтый для изменённых) и автоматическим выбором контрастного цвета текста, маркеры удалений показывают количество удалённых строк
- **Поиск и замена**: Поиск по тексту с поддержкой регистрозависимости и замена найденных совпадений
- **История изменений**: Отмена (Undo) и повтор (Redo) действий
- **Буфер обмена**: Копирование, вырезание и вставка через системный буфер обмена
- **Автоматическое сохранение**: Запрос на сохранение при закрытии файла с несохранёнными изменениями
- **Навигация по словам**: Перемещение курсора по словам через Ctrl+Left/Right, выделение по словам через Ctrl+Shift+Left/Right
- **Авто-отступ**: Новые строки автоматически наследуют отступ текущей строки; умный отступ добавляет дополнительный уровень после `{`, `(`, `[`, `:`
- **Авто-закрытие скобок**: Автоматическая вставка парных скобок и кавычек (`()`, `[]`, `{}`, `""`, `''`); ввод закрывающей скобки перешагивает через существующую; Backspace между парой удаляет оба символа

## Поиск и замена

### Интерактивный модал поиска (Ctrl+F)

Нажмите `Ctrl+F` для открытия интерактивного модала поиска с живым предпросмотром:

| Комбинация        | Действие                                   |
|-------------------|--------------------------------------------|
| `Ctrl+F`          | Открыть модал поиска                       |
| Ввод текста       | Поиск обновляется при вводе                |
| `Tab`             | Перейти к следующему совпадению            |
| `Shift+Tab`       | Перейти к предыдущему совпадению           |
| `F3`              | Перейти к следующему совпадению            |
| `Shift+F3`        | Перейти к предыдущему совпадению           |
| `Enter`           | Закрыть модал с выделенным совпадением     |
| `Escape`          | Закрыть модал поиска                       |
| Клик мыши         | Нажать кнопки навигации или `[X]` для закрытия |

**Возможности:**
- Живой предпросмотр поиска при вводе
- Счётчик совпадений (например, "3 из 12")
- Кнопки навигации: ◄ Назад, Вперёд ►
- Кнопка `[X]` для закрытия модала
- Запрос поиска сохраняется при закрытии модала

**Поведение поиска вне модала:**
- `F3` / `Shift+F3` - Навигация по совпадениям при закрытом модале
- `Tab` / `Shift+Tab` - Навигация по совпадениям когда поиск активен
- Любая клавиша навигации/редактирования - Деактивирует режим поиска
- Повторное открытие через `F3` восстанавливает последний запрос

### Интерактивный модал замены (Ctrl+H)

Нажмите `Ctrl+H` для открытия интерактивного модала замены с двумя полями ввода:

| Комбинация        | Действие                                   |
|-------------------|--------------------------------------------|
| `Ctrl+H`          | Открыть модал замены                       |
| Ввод в поле Найти | Поиск обновляется при вводе                |
| `Tab`             | След. совпадение (в Найти) или переход в поле Заменить |
| `Shift+Tab`       | Пред. совпадение (в Найти) или переход в поле Найти |
| `Up` / `Down`     | Навигация между полями Найти и Заменить    |
| `F3`              | Перейти к следующему совпадению            |
| `Shift+F3`        | Перейти к предыдущему совпадению           |
| `Enter`           | Заменить текущее и перейти к следующему    |
| `Escape`          | Закрыть модал замены                       |
| Клик мыши         | Нажать кнопки (Заменить, Всё, Назад, Вперёд) или `[X]` |

**Возможности:**
- Два поля ввода: Найти и Заменить
- Живой предпросмотр поиска при вводе в поле Найти
- Счётчик совпадений (например, "3 из 12")
- Четыре кнопки: Заменить, Всё, ◄ Назад, Вперёд ►
- Кнопка `[X]` для закрытия модала
- Оба текста (найти и заменить) сохраняются при закрытии модала

**Действия кнопок замены:**
- **Заменить** (`Ctrl+R`) - Заменить текущее совпадение и перейти к следующему
- **Всё** (`Ctrl+Alt+R`) - Заменить все совпадения, показать количество и закрыть модал
- **◄ Назад** - Перейти к предыдущему совпадению
- **Вперёд ►** - Перейти к следующему совпадению

**Обратная связь при замене всех:**
- После использования "Заменить всё" в статусной строке показывается количество замен
- Пример: "Заменено 5 вхождений"

## Поддержка мыши

- **Одиночный клик**: Установить курсор в позицию клика
- **Двойной клик**: Выделить слово под курсором
- **Тройной клик**: Выделить всю строку
- **Удержание + перемещение**: Выделение текста
- **Shift+клик** / **Alt+клик**: Расширить текущее выделение от якоря (или от курсора, если выделения ещё нет) до позиции клика. Alt+клик нужен потому, что GNOME Terminal / VTE перехватывает Shift+клик под собственное выделение текста; в kitty / alacritty / WezTerm работают оба варианта.
- **Колесо прокрутки**: Прокрутка содержимого редактора
- **Ctrl+Клик**: Перейти к определению (LSP); или показать превью цвета, если курсор находится на hex-значении (например, `#ff0000`, `#abc`)

**Превью цвета:** При удержании Ctrl и клике на hex-значении цвета появляется небольшой попап с цветным блоком и исходным кодом цвета. Попап отображается пока кнопка мыши удерживается и исчезает при её отпускании.

**Примечание:** Выделение мышью корректно работает в режиме переноса строк, учитывая перенесённые строки.

## Перенос строк (Word Wrap)

Когда включен режим переноса строк (настраивается в конфигурации), длинные строки автоматически переносятся по ширине панели. Редактор корректно обрабатывает:

- **Позиционирование курсора**: Навигация курсора и его отображение правильно работают на перенесённых строках
- **Выделение мышью**: Клики и перетаскивание точно выделяют текст даже когда строки занимают несколько визуальных рядов
- **Номера строк**: Отображаются для логических строк, а не визуальных рядов
- **Операции редактирования**: Все команды редактирования (вырезать, копировать, вставить, отменить/повторить) работают без проблем с перенесённым содержимым

Включить/выключить перенос строк в конфигурационном файле (`~/.config/termide/config.toml`):
```toml
[editor]
word_wrap = true  # или false
```

## Авто-отступ

Когда авто-отступ включён (по умолчанию), нажатие `Enter` создаёт новую строку, наследующую отступ текущей строки. Кроме того, умный отступ добавляет дополнительный уровень отступа после строк, заканчивающихся на `{`, `(`, `[` или `:`.

**Пример:**
```
if condition {|        ← курсор здесь, нажмите Enter
    |                  ← новая строка с доп. отступом
```

**Разделение скобок:** При нажатии `Enter` между парой скобок (`{|}`, `(|)`, `[|]`) редактор создаёт три строки — открывающая скобка, новая строка с отступом для курсора и закрывающая скобка на отдельной строке:

```
fn main() {|}          ← курсор между фигурными скобками, нажмите Enter
fn main() {
    |                  ← курсор здесь, с отступом
}
```

Включить/выключить авто-отступ в конфигурационном файле:
```toml
[editor]
auto_indent = true  # или false (по умолчанию: true)
```

## Авто-закрытие скобок

Когда авто-закрытие скобок включено (по умолчанию), ввод открывающей скобки или кавычки автоматически вставляет парный закрывающий символ, а курсор устанавливается между ними.

**Поддерживаемые пары:** `()`, `[]`, `{}`, `""`, `''`

**Поведение:**
- Ввод `(` вставляет `()` с курсором между ними
- Ввод `)`, когда символ под курсором уже `)`, перешагивает через него вместо вставки дубликата
- Нажатие `Backspace` между пустой парой (например, `()`) удаляет оба символа
- Кавычки не авто-закрываются после буквенно-цифровых символов (для поддержки апострофов в словах вроде "it's", "don't")

Включить/выключить авто-закрытие скобок в конфигурационном файле:
```toml
[editor]
auto_close_brackets = true  # или false (по умолчанию: true)
```

## Информация в статусной строке

При работе в редакторе статусная строка отображает:
- Имя файла и признак изменения (*)
- Текущую позицию курсора (строка:столбец)
- Размер табуляции (`Tab N`) — **кликабельно**: открывает маленькую модалку ввода, которая переопределяет `tab_size` только для текущей панели редактора. Override приоритетнее глобального `[editor].tab_size` и не затирается per-frame ресинком конфига, но `config.toml` не трогает. Принимает 1..=16; пустой/невалидный ввод игнорируется.
- Формат окончания строк (LF / CRLF) и кодировку (UTF-8)
- Информацию о поиске (количество совпадений)
- Тип файла (обычный текст / только чтение)

## Визуализация Git Diff

При редактировании файлов в git-репозитории с включённым `show_git_diff` редактор отображает информацию об изменениях в реальном времени по сравнению с HEAD:

### Цвета номеров строк

Номера строк используют фоновую подсветку для отображения статуса по сравнению с HEAD. Цвет текста подбирается автоматически для оптимальной контрастности:

- **Зелёный фон** - Строка была добавлена (отсутствует в HEAD)
- **Жёлтый фон** - Строка была изменена (отличается от HEAD)
- **Красный маркер (▶)** - Отмечает точку удаления (после этой строки были удалены строки)
- **Цвет по умолчанию** - Строка не изменена по сравнению с HEAD

### Маркеры удаления

Когда строки удалены, вставляется виртуальная строка для визуализации удаления:

- Отображается горизонтальная линия (`━`), охватывающая ширину редактора
- Показывается символ маркера удаления (`▶`) в области номера строки красным цветом
- Отображается текст по центру: "N строк удалено" (например, "3 строки удалены")
- Стилизовано серым/отключённым цветом для отличия от реального содержимого
- Не влияет на нумерацию строк (показывает `▶` вместо номера)

**Пример:**
```
  42 | function calculateTotal() {
 ▶   | ━━━━━━━ 5 строк удалено ━━━━━━━
  43 |     return result;
```

### Как это работает

- **Автоматические обновления**: Diff обновляется при сохранении файла
- **Сравнение в реальном времени**: Сравнивает текущее содержимое буфера с версией из HEAD
- **Поддержка отмены/повтора**: Маркеры появляются/исчезают при отмене/повторе удалений
- **Работает с редактированием**: Все обычные операции редактирования работают без проблем с визуализацией diff

### Конфигурация

Включить или отключить визуализацию git diff в конфигурационном файле (`~/.config/termide/config.toml`):

```toml
# Показывать цвета git diff на номерах строк (по умолчанию: true)
show_git_diff = true
```

**Примечания:**
- Работает только при редактировании файлов внутри git-репозитория
- Требует наличия файла в HEAD (новые неотслеживаемые файлы показывают все строки как добавленные)
- Виртуальные строки-маркеры удаления только визуальные и не влияют на содержимое файла

## Аннотации Git Blame

При редактировании файлов в git-репозитории редактор отображает встроенную аннотацию blame в конце строки под курсором, показывая автора, возраст, хеш коммита и краткое описание последнего изменения.

### Встроенная аннотация

Аннотация отображается в конце текущей строки приглушённым цветом:

```
  some_function();                          nvn, 2 недели назад • abc1234 Исправление
```

### Управление

Blame переключается через модал настроек: `Alt+P` → секция **Editor** → **Show Blame**.

**Примечание:** Blame **включён по умолчанию** при открытии файла в git-репозитории. Аннотация загружается асинхронно в фоне и появляется после завершения процесса `git blame`.

### Настройка

```toml
[editor]
show_blame = true  # или false (по умолчанию: true)
```

## LSP (Language Server Protocol)

TermIDE включает встроенную поддержку LSP для интеллектуальной помощи при написании кода. Когда языковой сервер настроен и доступен, вы получаете:

- **Автодополнение кода** — Контекстно-зависимые предложения при вводе
- **Диагностика** — Индикаторы ошибок и предупреждений в реальном времени
- **Статус загрузки** — Спиннер в заголовке панели показывает состояние сервера (запуск/индексация)

### Вызов автодополнения

| Комбинация        | Действие                                   |
|-------------------|--------------------------------------------|
| `Ctrl+J` / `Ctrl+Space` | Вручную вызвать всплывающее окно автодополнения |
| `Enter`           | Принять выбранное дополнение               |
| `Escape`          | Закрыть всплывающее окно                   |
| `↑` / `↓`         | Навигация по предложениям                  |
| Ввод символов     | Фильтрация предложений при вводе           |

**Автоматическое дополнение:** Когда включено (по умолчанию), всплывающее окно появляется автоматически:
- После ввода символов идентификаторов (буквы, цифры, `_`)
- Сразу после символов-триггеров (`.`, `:`, `(`, `<`)

### Всплывающее окно автодополнения

Во всплывающем окне отображаются:
- **Иконка** — Указывает тип элемента (функция ƒ, переменная v, класс C и т.д.)
- **Метка** — Текст дополнения
- **Индикаторы прокрутки** — ▲/▼ когда список прокручиваемый

### Конфигурация

Настройка LSP в файле конфигурации (`~/.config/termide/config.toml`):

```toml
[lsp]
enabled = true              # Включить/отключить LSP (по умолчанию: true)
auto_completion = true      # Автоматический вызов при вводе (по умолчанию: true)
completion_delay_ms = 100   # Задержка перед автодополнением (по умолчанию: 100)

[lsp.servers.rust]
command = "rust-analyzer"
args = []
root_markers = ["Cargo.toml"]

[lsp.servers.python]
command = "pylsp"
args = []
root_markers = ["pyproject.toml", "setup.py", "requirements.txt"]

[lsp.servers.typescript]
command = "typescript-language-server"
args = ["--stdio"]
root_markers = ["package.json", "tsconfig.json"]
```

### Действия с кодом (Code Actions)

| Клавиша            | Действие                                   |
|-------------------|--------------------------------------------|
| `Alt+Enter`       | Показать действия / быстрые исправления     |

Запрашивает быстрые исправления для текущей строки (например, «Import class» для добавления `use` в PHP) и показывает их во всплывающем списке. `↑`/`↓` — выбор, `Enter` — применить, `Esc` — закрыть. Правка выбранного действия применяется по файлам с перезагрузкой открытых редакторов. (`Ctrl+.` занят переключением комментария, поэтому по умолчанию используется `Alt+Enter`.)

### Индикатор состояния сервера

При открытии файла с поддержкой LSP заголовок панели показывает спиннер загрузки:
- `⠋ main.rs (starting)` — Сервер запускается
- `⠋ main.rs (indexing)` — Сервер индексирует проект
- `main.rs` — Сервер готов

### Поддерживаемые языки

LSP работает с любым языковым сервером, реализующим протокол LSP. Распространённые примеры:
- **Rust** — rust-analyzer
- **Python** — pylsp, pyright
- **TypeScript/JavaScript** — typescript-language-server
- **Go** — gopls
- **C/C++** — clangd

**Примечание:** Языковой сервер необходимо установить отдельно. TermIDE предоставляет только клиентскую интеграцию LSP.

## Режим Vim

TermIDE включает опциональный режим редактирования в стиле Vim с полной поддержкой кириллической раскладки.

### Включение режима Vim

Включите режим Vim в файле конфигурации (`~/.config/termide/config.toml`):

```toml
vim_mode = true
```

### Доступные режимы

| Режим | Описание | Индикатор |
|-------|----------|-----------|
| Normal | Навигация и команды | (нет) |
| Insert | Ввод текста | `-- INSERT --` |
| Visual | Посимвольное выделение | `-- VISUAL --` |
| Visual Line | Построчное выделение | `-- VISUAL LINE --` |

### Переключение режимов

| Клавиша | Действие |
|---------|----------|
| `Escape` | Вернуться в режим Normal |
| `i` | Вставка перед курсором |
| `I` | Вставка в начало строки |
| `a` | Добавление после курсора |
| `A` | Добавление в конец строки |
| `o` | Открыть строку ниже |
| `O` | Открыть строку выше |
| `v` | Войти в режим Visual |
| `V` | Войти в режим Visual Line |

### Клавиши перемещения (Normal/Visual)

| Клавиша | Действие |
|---------|----------|
| `h` / `←` | Переместиться влево |
| `j` / `↓` | Переместиться вниз |
| `k` / `↑` | Переместиться вверх |
| `l` / `→` | Переместиться вправо |
| `w` | Начало следующего слова |
| `b` | Начало предыдущего слова |
| `e` | Конец следующего слова |
| `0` | Начало строки |
| `$` | Конец строки |
| `^` | Первый непробельный символ |
| `gg` | Перейти к первой строке |
| `G` | Перейти к последней строке |
| `{число}G` | Перейти к номеру строки |

### Операторы (режим Normal)

| Клавиша | Действие |
|---------|----------|
| `d` | Удалить (+ перемещение) |
| `dd` | Удалить строку |
| `D` | Удалить до конца строки |
| `c` | Заменить (+ перемещение) |
| `cc` | Заменить строку |
| `C` | Заменить до конца строки |
| `y` | Копировать (+ перемещение) |
| `yy` | Копировать строку |
| `p` | Вставить после курсора |
| `P` | Вставить перед курсором |
| `x` | Удалить символ |
| `r` | Заменить символ |
| `u` | Отменить |
| `Ctrl+R` | Повторить |

### Поддержка кириллической раскладки

Режим Vim работает без проблем с кириллической раскладкой клавиатуры. При вводе на русском или других кириллических раскладках все команды Vim автоматически транслируются:

- `о` (русская) → `j` (переместиться вниз)
- `л` (русская) → `k` (переместиться вверх)
- `н` (русская) → `y` (копировать)
- `в` (русская) → `d` (удалить)

Это позволяет использовать команды Vim без переключения раскладки клавиатуры.

### Операции в режиме Visual

В режиме Visual или Visual Line:
- Используйте клавиши перемещения для расширения выделения
- `d` или `x` — Удалить выделение
- `y` — Копировать выделение
- `c` — Заменить выделение (удалить и войти в режим Insert)
- `>` — Увеличить отступ выделения
- `<` — Уменьшить отступ выделения

### Поиск в режиме Vim

| Клавиша | Действие |
|---------|----------|
| `/` | Поиск вперёд |
| `?` | Поиск назад |
| `n` | Следующее совпадение |
| `N` | Предыдущее совпадение |

**Примечание:** Стандартные сочетания клавиш редактора (`Ctrl+S`, `Ctrl+Z` и т.д.) продолжают работать во всех режимах Vim.
