# Файловый менеджер

Панель файлового менеджера предоставляет интуитивно понятный интерфейс для навигации по файловой системе и выполнения операций с файлами и каталогами.

Удалённые файловые системы (SFTP / FTP / FTPS) открываются в той же
панели, что и локальные пути — см. [Удалённые файловые
системы](vfs.md) для синтаксиса URL и настройки аутентификации.

## Навигация

| Комбинация        | Действие                                   |
|-------------------|--------------------------------------------|
| `↑` / `↓`         | Перемещение курсора вверх/вниз             |
| `Enter`           | Войти в каталог, просмотреть медиафайл или открыть файл |
| `Backspace`       | Перейти в родительский каталог             |
| `~`               | Перейти в домашний каталог                 |
| `PageUp` / `PageDown` | Прокрутка списка на одну страницу      |
| `Home` / `End`    | Перейти в начало/конец списка              |
| `.`               | Переключить видимость скрытых файлов       |
| `→` / `l`       | Развернуть каталог (древовидный вид)        |
| `←` / `h`       | Свернуть каталог (древовидный вид)          |
| `/`              | Инкрементальный поиск в дереве             |
| `Ctrl+/`          | Переключатель каталогов                    |
| `Ctrl+G`          | Перейти по пути/URL                        |
| `Alt+K`           | Добавить закладку                          |
| `Tab`             | Перейти на следующую панель                |
| `Shift+Tab`       | Перейти на предыдущую панель               |

## Выделение файлов

| Комбинация        | Действие                                   |
|-------------------|--------------------------------------------|
| `Insert`          | Выделить/снять выделение с текущего файла  |
| `Shift + ↑ / ↓`   | Выделить несколько файлов подряд           |
| `Ctrl+A`          | Выделить все файлы и каталоги в панели     |
| `Escape`          | Снять все выделения                        |

В древовидном виде выделение каталога через `Insert` каскадно выделяет все файлы внутри него. При сворачивании выделенного каталога выделение на его дочерних элементах сохраняется.

## Операции с файлами

| Комбинация        | Действие                                   |
|-------------------|--------------------------------------------|
| `Ctrl+N`          | Создать новый файл                         |
| `D` / `F7`        | Создать новый каталог                      |
| `Delete` / `F8`   | Удалить выделенные файлы/каталоги          |
| `C` / `F5`        | Копировать выделенные файлы/каталоги       |
| `M` / `F6`        | Переместить/переименовать файлы/каталоги   |
| `E` / `F4`        | Открыть файл в редакторе                   |
| `R` / `F2`        | Переименовать файл/каталог                 |
| `V` / `F3`        | Просмотр файла (без запуска)               |
| `Ctrl+R`          | Обновить содержимое текущего каталога      |
| `Пробел`          | Показать информацию о файле/каталоге       |

## Поиск

| Комбинация        | Действие                                   |
|-------------------|--------------------------------------------|
| `Ctrl+F`          | Поиск файлов по имени (glob-шаблоны)       |
| `Ctrl+Shift+F`    | Поиск/замена по содержимому файлов         |
| `/`              | Инкрементальный поиск в дереве (фильтрация при вводе) |

Оба поиска используют **встроенную полоску, докнутую сверху панели** (не
плавающий модал): форма и результаты делят панель, а по совпадениям можно
ходить курсором при открытой форме. `Tab` / стрелки перемещают фокус между
полями и кнопками; `Esc` закрывает полоску.

### Поиск файлов (Ctrl+F)

Открывает встроенную полоску с одним glob-полем для быстрого поиска по имени:
- Фильтрация файлов в реальном времени при вводе; результаты с цветами git
  показываются под полоской.
- `Enter` переводит фокус в список результатов, затем `↑` / `↓` двигают курсор,
  а `Enter` открывает выбранный файл. `Tab` возвращает в поле, `Escape`
  закрывает.

### Поиск и замена по содержимому (Ctrl+Shift+F)

Открывает встроенную полоску с полями `Mask:` / `Find:` / `Repl:` для поиска
текста внутри файлов:
- По умолчанию поиск **литеральный**; тумблер `[.*] Regex` включает регулярные
  выражения, `[Aa] Case` — учёт регистра. Переключаются кликом либо фокусом на
  ряду кнопок и нажатием `Enter` / `Space`.
- Поиск только в текстовых файлах (бинарные пропускаются); большие файлы
  пропускаются (настраиваемый лимит в конфигурации).
- Результаты **сгруппированы по файлам** под полоской: строка-заголовок с числом
  совпадений, затем по одной строке на совпадение (номер строки + строка с
  подсветкой). `Enter` на поле `Find` переводит фокус в результаты, затем `↑` /
  `↓` навигируют, а `Enter` открывает файл на этой строке; `Escape` закрывает.

**Замена по файлам.** Введите замену в поле `Repl:`. Совпадение под курсором
показывает предпросмотр `-старое/+новое`. `Enter` в поле `Repl:` (или кнопка
`All`) заменяет **все** совпадения во всех файлах из списка — после подтверждения
с числом вхождений и файлов. При включённом `[.*] Regex` в замене работают
группы захвата `$1` / `${name}`; иначе текст вставляется буквально. Замены
пишутся на диск.

### Поиск в дереве (/)

Нажмите `/` для инкрементального поиска в текущем дереве каталогов:
- Фильтрует список файлов по мере ввода, показывая только совпадения
- Родительские каталоги совпавших файлов остаются видимыми для контекста
- Нажмите `Enter` для подтверждения и перехода к совпадению
- Нажмите `Escape` для отмены и восстановления полного дерева
- Работает совместно с древовидным видом — каталоги с совпадениями автоматически разворачиваются

## Буфер обмена

| Комбинация        | Действие                                   |
|-------------------|--------------------------------------------|
| `Ctrl+C`          | Копировать пути выделенных элементов       |
| `Ctrl+X`          | Вырезать пути выделенных элементов         |
| `Ctrl+V`          | Вставить файлы из буфера обмена            |

### Правило целевой директории

Создание файла, создание каталога и вставка из буфера обмена
направляются **на уровень курсора**, а не всегда в корень панели:

- Курсор на записи верхнего уровня → действие выполняется в текущей
  директории панели.
- Курсор внутри развёрнутого подкаталога → действие выполняется в этом
  подкаталоге.

Это соответствует визуальному положению курсора — где видно, там и
создаётся или вставляется файл.

## Интеграция с Git

Файловый менеджер отображает статус файлов в Git-репозиториях:

- **Цветовая индикация статуса** — новые, изменённые, удалённые и неотслеживаемые файлы выделены цветом
- **Вложенный статус** — каталоги показывают агрегированный статус дочерних элементов (например, каталог с изменёнными файлами подсвечивается)
- **Интеграция с деревом** — статус git распространяется по дереву каталогов, упрощая поиск изменений в глубоких иерархиях

## Просмотр медиафайлов

Файловый менеджер может просматривать изображения и видео с помощью консольных просмотрщиков.

**Логика открытия файлов:**

| Тип файла | Действие |
|-----------|----------|
| Растровые изображения (PNG, JPG, JPEG, GIF, WebP, BMP, TIFF) | ImagePanel (нативная графика) или xdg-open |
| Векторные изображения (SVG, ICO) | xdg-open (системный просмотрщик) |
| Видео (MP4, MKV, AVI, MOV, WebM, FLV, WMV, M4V) | xdg-open (системный плеер) |
| Бинарные файлы | xdg-open (системное приложение) |
| Текстовые файлы | Панель редактора |
| Исполняемые файлы | Запуск в терминале |

**Горячие клавиши:**
- `Enter` → умное открытие (см. таблицу выше)
- `F3` → просмотр файла (как Enter, но исполняемые открываются в редакторе)
- `O` / `Alt+Enter` → принудительно открыть через xdg-open (системное приложение)
- `F4` → всегда открывать в редакторе

**Нативная графика:**
termide автоматически определяет, поддерживает ли родительский терминал графические протоколы (Kitty, Sixel, iTerm2). При наличии поддержки растровые изображения рендерятся напрямую в ImagePanel без внешних инструментов.

**Поддерживаемые терминалы:**
- Kitty, WezTerm, iTerm2, Ghostty, foot - полная поддержка графики
- Другие терминалы - fallback на xdg-open

## Поддержка мыши

- **Одиночный клик**: Выбрать файл или каталог.
- **Двойной клик**: Войти в каталог или открыть файл.
- **Колесо прокрутки**: Прокрутка списка файлов.

## Отображение

### Модификаторы шрифта

- **Курсив** — символические ссылки (файлы и каталоги)
- **Жирный** — исполняемые файлы

### Порядок сортировки

Файлы отображаются группами, каждая отсортирована по алфавиту:
1. Каталоги
2. Исполняемые файлы
3. Обычные файлы

### Символические ссылки

- Модальное окно копирования содержит флажок «Создать символическую ссылку» — создаёт ссылку вместо копирования
- Когда включён флажок «Создать символическую ссылку», появляется флажок «Использовать относительный путь», и новая ссылка указывает на относительный путь вместо абсолютного
- При переходе в каталог-ссылку файловый менеджер следует по ссылке в целевой каталог
- `Space` на симлинке открывает модальное окно свойств со строкой **Цель**, показывающей то, что реально записано в ссылке (без `canonicalize`), с сокращением домашнего каталога до `~`
- В статусной строке после имени выделенного симлинка показывается `→ цель`; работает и для симлинков во вложенных подкаталогах развёрнутого tree view

### Примечания

- `..` (родительский каталог) не может быть выделен — используется только для навигации
- Модальные окна ввода поддерживают раскрытие `~/` в домашний каталог

## Конфигурация

Файловый менеджер читает свои настройки из секции `[file_manager]` в
`config.toml`. Самые заметные опции:

| Ключ                                | По умолчанию | Описание                                                                                                       |
|-------------------------------------|--------------|----------------------------------------------------------------------------------------------------------------|
| `extended_view_width`               | `50`         | Минимальная ширина панели (столбцов), при которой extended view показывает столбцы размера и времени.          |
| `content_search_max_file_size_mb`   | `1`          | Максимальный размер файла, рассматриваемого `Ctrl+Shift+F`. Файлы крупнее пропускаются.                        |
| `dir_size_in_wide_view`             | `true`       | Считать и показывать размеры каталогов в колонке Size extended view. Только для локальной ФС.                  |
| `dir_size_budget_ms`                | `100`        | Бюджет времени (мс) на обход одного каталога. Не уложившиеся в бюджет деревья рисуются как `-`. `0` — выключено. |

Обходы используют общий процесс-глобальный кэш, поэтому две панели
с одинаковой директорией не делают двойной walk. Нажатие `Space` на
каталоге также публикует его точный (полный) размер в этот кэш —
колонка размера в wide view подхватит результат на следующем redraw.
