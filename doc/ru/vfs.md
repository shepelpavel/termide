# Удалённые файловые системы (VFS)

TermIDE содержит чистый Rust-слой VFS, позволяющий [файловому
менеджеру](file-manager.md) просматривать удалённые серверы как
обычные локальные каталоги. Файлы можно открывать в редакторе,
копировать между локальной и удалённой панелями, переименовывать и
т.д. — тот же интерфейс, с прогрессом в [панели
операций](operations.md).

Системные библиотеки не требуются: SFTP работает на `russh` +
`russh-sftp`, FTPS — на `rustls`. Статические сборки под Alpine / musl
работают «из коробки».

## Поддерживаемые протоколы

| Схема     | Протокол                                            |
|-----------|-----------------------------------------------------|
| `sftp://` | SSH File Transfer Protocol поверх SSH               |
| `ftp://`  | Обычный FTP                                         |
| `ftps://` | FTP поверх TLS (rustls)                             |

`smb://` и `nfs://` распознаются парсером URL, но соответствующих
провайдеров пока нет в поставке — используйте нативное монтирование
ОС.

## Синтаксис URL

VFS URL имеет вид:

```
scheme://[user[:password]@]host[:port][/path]
```

Примеры:

```
sftp://nvn@example.com/home/nvn/projects
sftp://example.com:2222/srv/builds
ftp://files.example.com/pub
ftps://secure.example.com/uploads
```

Замечания:
- Не-ASCII символы в `path` поддерживаются; парсер percent-decode'ит
  их обратно в UTF-8 перед отправкой на сервер.
- Если пользователь опущен, провайдер SFTP подставляет значение из
  SSH config (см. Аутентификацию).
- Если порт опущен, используется значение по умолчанию (22 / 21 / 990).
- Встраивание пароля в URL поддерживается, но не рекомендуется —
  для SFTP предпочтительна аутентификация по ключу.

## Открытие удалённой локации

Два пути к удалённой панели:

1. **Go to path** — открыть в файловом менеджере поле ввода
   `Go to path` и вставить / ввести VFS URL.
2. **Bookmarks** — сохранить часто используемые удалённые точки и
   возвращаться к ним через меню `Bookmarks`. Записи закладок
   принимают тот же URL-синтаксис; хранятся в
   `~/.config/termide/bookmarks.toml` (или в `.termide/bookmarks.toml`
   проекта, если нужны проектные shortcut'ы).

В обоих случаях открывается обычная панель файлового менеджера в
удалённом каталоге; всё остальное (раскрытие дерева, копирование,
переименование, свойства по `Space`) полностью совпадает с поведением
локальных панелей.

## Аутентификация (SFTP)

SFTP поддерживает четыре режима аутентификации:

- **`Auto`** *(по умолчанию)* — пробует сначала SSH agent, затем ключи
  из `~/.ssh/config` для этого хоста (включая `IdentityFile`, `User`,
  `Port`, `Hostname`-алиасы), затем дефолтные ключи (`id_ed25519`,
  `id_rsa`, `id_ecdsa`, `id_dsa`), затем пароль.
- **SSH agent** — использует SSH agent, если `SSH_AUTH_SOCK` указывает
  на работающий.
- **SSH key** — явный приватный ключ, опционально с парольной фразой.
- **Password** — интерактивный запрос или предварительно сохранённое
  значение.

Поскольку `Auto` читает `~/.ssh/config`, в bookmark можно держать
короткий URL (`sftp://my-build-host/path`) и позволить SSH config
подставить настоящие host, user, ключ. Та же конфигурация, что
использует CLI `ssh`.

## Поведение удалённых панелей

Все удалённые операции асинхронны:

- Первое чтение каталога показывает короткий спиннер; записи
  появляются как только сервер ответит.
- Раскрытие подкаталога вставляет маленький `…`-плейсхолдер; когда
  листинг придёт, он заменяется реальными детьми.
- Состояние дерева, курсор и выделение сохраняются между перезагрузками.

Размеры каталогов в wide view **не** считаются для удалённых панелей —
обход удалённого дерева ради одного столбца дороже, чем выгода.
Колонка размера для remote всегда пустая.

## Передача файлов

Выберите файлы в одной панели, нажмите `C` / `F5` (копирование) или
`M` / `F6` (перемещение), укажите назначение — локальный путь или
другую удалённую панель. Передача регистрируется как операция в
[панели операций](operations.md) со следующим:

- Реальный progress bar (байты + файлы), обновляемый воркером на
  каждый chunk.
- Pause / Resume действительно останавливает поток байт (paused
  uploads / downloads висят на worker'е; актор SFTP остаётся свободен
  для metadata-запросов других панелей).
- Cancel останавливает чисто между chunk'ами. Если осталась
  частичная копия на сервере, панель предложит её удалить — см.
  раздел очистки после отмены в [панели операций](operations.md).

Переименование внутри одного SFTP / FTP-соединения остаётся на
сервере: move в пределах одного хоста выполняется как remote-rename,
а не download-then-upload.

## Сессии

Сессия с удалённой панелью сохраняет URL точно так же, как для
локального пути. При следующем запуске TermIDE переподключается в
фоне и показывает панель с загрузочным placeholder'ом до прихода
листинга — остальной UI отзывается сразу.

### Обрыв соединения

Если удалённая сессия потеряна (таймаут простоя, обрыв сети),
следующая операция завершается ошибкой, и панель показывает диалог
восстановления с тремя вариантами: **Переподключиться** (свежая сессия
к тому же пути), **Открыть домашний (локально)** (сбросить соединение и
переключить панель на локальный домашний каталог) или **Закрыть
панель**. `Esc` закрывает диалог, оставляя панель на последнем листинге.
Бесконечного цикла ошибки больше нет.

## Ограничения / известные пробелы

- Провайдеры `smb://` / `nfs://` пока не реализованы — только разбор
  URL.
- Нет резюма прерванной передачи с offset'a: отменённые upload'ы при
  повторе начинаются сначала.
- Интерактивный пароль не сохраняется в TermIDE; для постоянного
  использования настройте ключи SSH или SSH agent.
