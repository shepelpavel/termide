# Пользовательские действия (Actions)

Система Actions позволяет добавлять пользовательские скрипты в меню TermIDE. Скрипты выполняются в новой панели терминала, что упрощает запуск команд сборки, скриптов развёртывания или любых задач автоматизации прямо из TermIDE.

## Начало работы

### Расположение каталога Actions

Разместите скрипты в каталоге actions:

| Платформа | Путь |
|-----------|------|
| Linux | `~/.config/termide/actions/` |
| macOS | `~/Library/Application Support/termide/actions/` |
| Windows | `%APPDATA%\termide\actions\` |

Вы также можете открыть эту папку через `Options → Manage actions` в меню.

### Создание первого действия

```bash
# Создаём каталог actions
mkdir -p ~/.config/termide/actions

# Создаём простой скрипт
cat > ~/.config/termide/actions/hello.sh << 'EOF'
#!/bin/bash
echo "Привет из TermIDE!"
echo "Текущий каталог: $(pwd)"
read -p "Нажмите Enter для закрытия..."
EOF

# Делаем скрипт исполняемым (обязательно для Unix)
chmod +x ~/.config/termide/actions/hello.sh
```

После создания скрипта перезапустите TermIDE или используйте `Options → Manage actions` для обновления. Ваш скрипт появится в меню **Actions**.

## Именование скриптов

Отображаемое имя в меню формируется из имени файла:

| Имя файла | Отображаемое имя |
|-----------|------------------|
| `build.sh` | build |
| `deploy.sh` | deploy |
| `run-tests.py` | run-tests |
| `my.cool.script.sh` | my |

Отображаемое имя — это часть имени файла до первой точки.

## Структура каталогов (группы)

Скрипты можно организовать в группы с помощью подкаталогов. Каждый подкаталог становится подменю:

```
~/.config/termide/actions/
├── build.sh              # Появляется в корне меню Actions
├── deploy.sh             # Появляется в корне меню Actions
├── docker/               # Создаёт подменю "docker"
│   ├── up.sh
│   ├── down.sh
│   └── logs.sh
└── git/                  # Создаёт подменю "git"
    ├── pull.sh
    ├── push.sh
    └── status.sh
```

**Примечание:** Поддерживается только один уровень вложенности. Вложенные подкаталоги игнорируются.

## Фоновое выполнение

По умолчанию скрипты запускаются в панели терминала на переднем плане. Для долго выполняющихся процессов, которые нужно запустить в фоне, добавьте `.bg.` в имя файла:

| Имя файла | Режим выполнения |
|-----------|------------------|
| `server.sh` | Передний план (новая панель терминала) |
| `server.bg.sh` | Фоновый режим |
| `deploy.bg.sh` | Фоновый режим |

Фоновые скрипты запускаются без открытия панели терминала, что полезно для:
- Запуска серверов разработки
- Запуска процессов наблюдения (watch)
- Запуска фоновых служб

## Особенности платформ

### Unix (Linux/macOS)

Скрипты должны иметь права на выполнение:

```bash
chmod +x ~/.config/termide/actions/myscript.sh
```

Любой файл с установленным битом выполнения появится в меню, независимо от расширения.

### Windows

В Windows распознаются следующие расширения файлов:
- `.sh` (требуется WSL или Git Bash)
- `.bat`
- `.cmd`
- `.ps1` (PowerShell)
- `.py` (Python)
- `.rb` (Ruby)
- `.pl` (Perl)

## Рабочий каталог

Скрипты выполняются с корневым каталогом текущей сессии в качестве рабочего каталога. Обычно это каталог, из которого был запущен TermIDE, или каталог, выбранный через `Sessions → Change root`.

## Советы

1. **Добавляйте shebang**: Всегда начинайте скрипты с shebang (например, `#!/bin/bash`), чтобы они запускались правильным интерпретатором.

2. **Сохраняйте вывод видимым**: Для скриптов переднего плана добавьте `read -p "Нажмите Enter..."` в конце, чтобы увидеть вывод до закрытия терминала.

3. **Используйте понятные имена**: Имя файла до первой точки становится меткой в меню, поэтому используйте понятные описательные имена.

4. **Организуйте с помощью групп**: Используйте подкаталоги для группировки связанных скриптов (например, `docker/`, `npm/`, `git/`).

5. **Фоновый режим для серверов**: Используйте `.bg.` в имени файла для скриптов, запускающих долго работающие процессы.

## Устранение неполадок

### Скрипт не появляется в меню

1. Проверьте, что файл имеет права на выполнение (`chmod +x`)
2. Убедитесь, что файл находится в правильном каталоге
3. Перезапустите TermIDE для обновления списка действий

### Скрипт не запускается

1. Сначала протестируйте скрипт вручную в терминале
2. Проверьте правильность строки shebang
3. Убедитесь, что все необходимые инструменты/интерпретаторы установлены

### Фоновый скрипт не работает

Фоновые скрипты выполняются беззвучно. Проверьте:
1. Скрипт действительно запускает нужный процесс
2. Процесс не завершается сразу после запуска
3. При необходимости ищите вывод процесса в системных журналах
