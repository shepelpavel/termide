# Пользовательские скрипты

> **Примечание:** этот документ описывает **систему пользовательских скриптов**. Клавиатурные сочетания и действия интерфейса — в [ui.md](ui.md#Клавиатурная-навигация-и-управление-по-меню-и-панелям).

Система скриптов позволяет добавлять пользовательские скрипты в меню TermIDE. Скрипты выполняются в новой панели терминала, что упрощает запуск команд сборки, скриптов развёртывания или любых задач автоматизации прямо из TermIDE.

## Начало работы

### Расположение каталога скриптов

Разместите скрипты в каталоге scripts:

| Платформа | Путь |
|-----------|------|
| Linux | `~/.local/share/termide/scripts/` |
| macOS | `~/Library/Application Support/termide/scripts/` |
| Windows | `%APPDATA%\termide\scripts\` |

Вы также можете открыть эту папку через `Options → Управление скриптами` в меню.

### Создание первого скрипта

```bash
# Создаём каталог скриптов
mkdir -p ~/.local/share/termide/scripts

# Создаём простой скрипт
cat > ~/.local/share/termide/scripts/hello.sh << 'EOF'
#!/bin/bash
echo "Привет из TermIDE!"
echo "Текущий каталог: $(pwd)"
read -p "Нажмите Enter для закрытия..."
EOF

# Делаем скрипт исполняемым (обязательно для Unix)
chmod +x ~/.local/share/termide/scripts/hello.sh
```

После создания скрипта перезапустите TermIDE или используйте `Options → Управление скриптами` для обновления. Ваш скрипт появится в меню **Скрипты**.

## Именование скриптов

Отображаемое имя в меню формируется из имени файла:

| Имя файла | Отображаемое имя |
|-----------|------------------|
| `build.sh` | build |
| `deploy.sh` | deploy |
| `run-tests.py` | run-tests |
| `my.cool.script.sh` | my |

Отображаемое имя — это часть имени файла до первой точки.

## Структура каталогов (группы)

Скрипты можно организовать в группы с помощью подкаталогов. Каждый подкаталог становится подменю:

```
~/.local/share/termide/scripts/
├── build.sh              # Появляется в корне меню Скрипты
├── deploy.sh             # Появляется в корне меню Скрипты
├── docker/               # Создаёт подменю "docker"
│   ├── up.sh
│   ├── down.sh
│   └── logs.sh
└── git/                  # Создаёт подменю "git"
    ├── pull.sh
    ├── push.sh
    └── status.sh
```

**Примечание:** Поддерживается только один уровень вложенности. Вложенные подкаталоги игнорируются.

## Режимы выполнения

Скрипты поддерживают различные режимы выполнения в зависимости от суффиксов в имени файла:

### Фоновое выполнение (`.bg.`)

Для долго выполняющихся процессов, которые нужно запустить в фоне, добавьте `.bg.` в имя файла:

| Имя файла | Режим выполнения |
|-----------|------------------|
| `server.sh` | Передний план (новая панель терминала) |
| `server.bg.sh` | Фоновый режим (без панели терминала) |
| `deploy.bg.sh` | Фоновый режим |

Фоновые скрипты запускаются без открытия панели терминала, что полезно для:
- Запуска серверов разработки
- Запуска процессов наблюдения (watch)
- Запуска фоновых служб

### Скрипты с отчётом (`.report.`)

Для скриптов, которые должны выполняться в фоне и показывать вывод в модальном окне, добавьте `.report.` в имя файла:

| Имя файла | Режим выполнения |
|-----------|------------------|
| `check.sh` | Передний план (новая панель терминала) |
| `check.report.sh` | Фон с выводом в модальное окно |
| `status.report.sh` | Фон с выводом в модальное окно |

Скрипты с отчётом:
- Выполняются в фоне, не блокируя интерфейс
- Захватывают stdout и stderr
- Отображают вывод в информационном модальном окне по завершении
- Показывают индикатор успеха (✓) или ошибки (✗) в заголовке окна
- Поддерживают **скроллинг**, если вывод не умещается в окно

**Навигация в окне отчёта:**

| Комбинация | Действие |
|-----------|----------|
| `↑` / `↓` | Прокрутить на одну строку |
| `PageUp` / `PageDown` | Прокрутить на высоту видимой области |
| `Home` / `End` | В начало / в конец |
| Колесо мыши | Плавная прокрутка (±3 строки) |
| `Enter` / `Escape` | Закрыть окно |

Справа на рамке окна отображается **индикатор позиции** (`▌` на линии `│`), если содержимое превышает высоту области.

**Примеры использования:**
- Быстрые проверки состояния (`git status`, `docker ps`)
- Скрипты линтинга или валидации
- Проверки состояния системы
- Любые короткие скрипты, результат которых нужно увидеть

**Пример:**
```bash
# Создаём скрипт с отчётом
cat > ~/.local/share/termide/scripts/check.report.sh << 'EOF'
#!/bin/bash
echo "Проверка состояния системы..."
echo "Дата: $(date)"
echo "Пользователь: $(whoami)"
echo "PWD: $(pwd)"
EOF
chmod +x ~/.local/share/termide/scripts/check.report.sh
```

## Особенности платформ

### Unix (Linux/macOS)

Скрипты должны иметь права на выполнение:

```bash
chmod +x ~/.local/share/termide/scripts/myscript.sh
```

Любой файл с установленным битом выполнения появится в меню, независимо от расширения.

### Windows

В Windows распознаются следующие расширения файлов:
- `.sh` (требуется WSL или Git Bash)
- `.bat`
- `.cmd`
- `.ps1` (PowerShell)
- `.py` (Python)
- `.rb` (Ruby)
- `.pl` (Perl)

## Рабочий каталог

Скрипты выполняются с корневым каталогом текущей сессии в качестве рабочего каталога. Обычно это каталог, из которого был запущен TermIDE, или каталог, выбранный через `Sessions → Change root`.

## Советы

1. **Добавляйте shebang**: Всегда начинайте скрипты с shebang (например, `#!/bin/bash`), чтобы они запускались правильным интерпретатором.

2. **Сохраняйте вывод видимым**: Для скриптов переднего плана добавьте `read -p "Нажмите Enter..."` в конце, чтобы увидеть вывод до закрытия терминала.

3. **Используйте понятные имена**: Имя файла до первой точки становится меткой в меню, поэтому используйте понятные описательные имена.

4. **Организуйте с помощью групп**: Используйте подкаталоги для группировки связанных скриптов (например, `docker/`, `npm/`, `git/`).

5. **Фоновый режим для серверов**: Используйте `.bg.` в имени файла для скриптов, запускающих долго работающие процессы.

6. **Отчёт для быстрых проверок**: Используйте `.report.` в имени файла для скриптов, результат которых нужно увидеть в модальном окне.

## Устранение неполадок

### Скрипт не появляется в меню

1. Проверьте, что файл имеет права на выполнение (`chmod +x`)
2. Убедитесь, что файл находится в правильном каталоге
3. Перезапустите TermIDE для обновления списка скриптов

### Скрипт не запускается

1. Сначала протестируйте скрипт вручную в терминале
2. Проверьте правильность строки shebang
3. Убедитесь, что все необходимые инструменты/интерпретаторы установлены

### Фоновый скрипт не работает

Фоновые скрипты выполняются беззвучно. Проверьте:
1. Скрипт действительно запускает нужный процесс
2. Процесс не завершается сразу после запуска
3. При необходимости ищите вывод процесса в системных журналах

### Модальное окно скрипта с отчётом не появляется

Скрипты с отчётом показывают модальное окно по завершении:
1. Дождитесь завершения выполнения скрипта
2. Проверьте, не завершается ли скрипт слишком быстро
3. Убедитесь, что скрипт выводит данные в stdout или stderr
