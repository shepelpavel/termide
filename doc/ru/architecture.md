# Архитектура

Этот документ описывает техническую архитектуру TermIDE.

## Общий обзор

TermIDE — это терминальная IDE, построенная на Rust с использованием TUI-фреймворка `ratatui`. Она использует адаптивную **компоновку с вертикально разделяемыми колонками панелей**: каждая панель в столбце имеет независимо регулируемую высоту, плюс однонажимный пресет полноэкранного режима для активной панели.

```
┌─────────────────────────────────────────────────────────┐
│ Строка меню  [CPU] [RAM] [Часы]                         │
├───────────────────┬─────────────────────────────────────┤
│ ┌[≡] 📁 Файлы ──┐ │ ┌[≡] 📝 Редактор: main.rs ────────┐│
│ │ src/          │ │ │                                  ││
│ │ tests/        │ │ │  fn main() {                     ││
│ │ Cargo.toml    │ │ │      // код здесь                ││
│ │               │ │ │  }                               ││
│ ├[≡] 💻 Терминал┤ │ │                                  ││
│ │ $ cargo build │ │ │                                  ││
│ │ Сборка...     │ │ │                                  ││
│ └───────────────┘ │ └──────────────────────────────────┘│
├───────────────────┴─────────────────────────────────────┤
│ Статус: file.rs:42  Стр 10, Кол 5       Диск: 83%      │
└─────────────────────────────────────────────────────────┘
```

В левой колонке выше — две панели с собственными высотами; справа — одна панель занимает всю колонку. Нажатие `Alt+F11` сворачивает все нефокусные панели в активной колонке до строки заголовка (пресет на одну строку), повторное нажатие восстанавливает прежние высоты.

## Основные архитектурные компоненты

### 1. Система компоновки

#### 1.1 LayoutManager

**Расположение:** `crates/layout/src/lib.rs`

`LayoutManager` владеет состоянием split-компоновки. Он управляет:

**Компоненты:**
- `panel_groups: Vec<PanelGroup>` - Горизонтальное расположение групп панелей
- `focus: usize` - Текущий фокус (индекс активной группы панелей)

**Ключевые обязанности:**
- Добавление панелей с автоматическим стекированием на основе порога ширины
- Управление горизонтальной навигацией (Alt+Left/Right)
- Управление вертикальным фокусом внутри группы (Alt+Up/Down)
- Умное стекирование/разделение панелей между группами (Alt+Backspace, F11)
- Пропорциональное перераспределение ширин при ресайзе терминала
- Закрытие панелей и очистка пустых групп

**Управление фокусом:**
Фокус — это простой индекс `usize`, указывающий, какая группа панелей в данный момент активна. Фокусная группа получает ввод с клавиатуры/мыши и подсвечивается в UI.

#### 1.2 PanelGroup

**Расположение:** `crates/layout/src/panel_group.rs`

`PanelGroup` представляет вертикальный split-стек панелей в одной колонке.

**Структура:**
```rust
pub struct PanelGroup {
    panels: Vec<Box<dyn Panel>>,         // Панели в этой группе
    expanded_index: usize,               // Фокусная панель (активная рамка)
    pub width: Option<u16>,              // Ширина колонки (None = авто-распределение)
    split_heights: Option<Vec<u16>>,     // Кешированные высоты панелей
    fullscreen_cache: Option<Vec<u16>>,  // Высоты до входа в fullscreen-пресет
}
```

**Поведение компоновки:**
- Все панели в группе видны. Минимальная высота каждой — одна строка (заголовок).
- `split_heights` — кеш per-panel высот. `None` означает «нет кеша — равномерное распределение при первом обращении». Высоты пропорционально пересчитываются при ресайзе.
- `Alt+F11` (action `toggle_fullscreen_panel`) переключает fullscreen-пресет: фокусная панель занимает всю высоту колонки, остальные сжимаются до одной строки. Высоты до пресета сохраняются в `fullscreen_cache` для restore при повторном нажатии.
- В пресете переключение фокуса через `Alt+Up`/`Alt+Down` (`prev_panel`/`next_panel`) переприменяет пресет к новому фокусу — визуально как старый аккордеон.
- `Ctrl+Alt+=` / `Ctrl+Alt+-` (`panel_grow_vertical` / `panel_shrink_vertical`) увеличивают/уменьшают фокусную панель на 3 строки, забирая место у соседа с запасом (cascade).

**Ключевые операции:**
- `add_panel()` / `insert_panel()` / `remove_panel()` — добавление/удаление с ребалансировкой кешей высот.
- `set_expanded()` / `next_panel()` / `prev_panel()` — перемещают фокус; переприменяют пресет, если он активен.
- `toggle_fullscreen()` — включение/выключение пресета.
- `grow_focused()` / `shrink_focused()` — изменение высоты фокусной панели.
- `resize_panel_divider()` — применение дельты к разделителю над указанной панелью (используется обработчиком mouse drag).

#### 1.3 Автоматическое стекирование

При добавлении новой панели через `LayoutManager::add_panel()`:

```rust
let new_width_if_split = available_width / (num_groups + 1);

if new_width_if_split < config.min_panel_width {
    // Стекировать вертикально в активную группу (split с кешем высот)
    active_group.add_panel(panel);
} else {
    // Создать новую горизонтальную группу
    let new_group = PanelGroup::new(panel);
    panel_groups.push(new_group);
}
```

**Порог по умолчанию:** `min_panel_width = 80` символов

Это гарантирует, что панели всегда имеют достаточно места для использования.

### 2. Система панелей

#### 2.1 Трейт Panel

**Расположение:** `crates/core/src/lib.rs`

Все панели реализуют трейт `Panel`, который определяет интерфейс для интерактивных терминальных панелей:

```rust
pub trait Panel {
    /// Отрисовать содержимое панели
    fn render(
        &mut self,
        area: Rect,                // Доступная область рендеринга
        buf: &mut Buffer,          // Буфер ratatui
        is_focused: bool,          // Имеет ли эта панель фокус?
        panel_index: usize,        // Индекс панели для идентификации
        state: &AppState,          // Общее состояние приложения
    );

    /// Обработать ввод с клавиатуры
    fn handle_key(&mut self, key: KeyEvent) -> Result<()>;

    /// Обработать ввод мыши
    fn handle_mouse(&mut self, mouse: MouseEvent, panel_area: Rect) -> Result<()>;

    /// Получить заголовок панели (показывается в шапке)
    fn title(&self) -> String;

    /// Проверить, является ли это панелью приветствия (автоматически закрывается при открытии других)
    fn is_welcome_panel(&self) -> bool { false }

    /// Получить файл для открытия (для панелей, запрашивающих открытие файла)
    fn take_file_to_open(&mut self) -> Option<PathBuf> { None }

    /// Получить рабочую директорию для новых панелей
    fn get_working_directory(&self) -> Option<PathBuf> { None }

    /// Получить запрос модального окна (для панелей, открывающих модальные окна)
    fn take_modal_request(&mut self) -> Option<(PendingAction, ActiveModal)> { None }
}
```

### 3. Обработка событий

#### 3.1 Цикл событий

**Расположение:** `crates/app/src/app/mod.rs`

Структура основного цикла событий:

```rust
while !state.should_quit {
    match event_handler.next()? {
        Event::Key(key) => self.handle_key_event(key)?,
        Event::Mouse(mouse) => self.handle_mouse_event(mouse)?,
        Event::Resize(w, h) => state.update_terminal_size(w, h),
        Event::Tick => {
            // Периодические обновления
            self.update_panels_tick()?;
            self.system_monitor.update(&mut self.state);
        }
    }
    self.render(terminal)?;
}
```

**Типы событий:**
- **Key** - Ввод с клавиатуры (горячие клавиши, ввод текста)
- **Mouse** - Клики мыши, перетаскивание, прокрутка
- **Resize** - Изменение размера терминала
- **Tick** - Периодический таймер (мониторинг ресурсов, обновление панелей)

#### 3.2 Обработчик клавиш

**Расположение:** `crates/app/src/app/key_handler.rs`

Обрабатывает ввод с клавиатуры с приоритетом:

1. **Модальное окно захватывает ввод первым** (если открыто)
2. **Глобальные горячие клавиши** (Alt+M, Alt+H, Alt+Q и т.д.)
3. **Управление панелями** (Alt+Left/Right, Alt+Up/Down, Alt+X и т.д.)
4. **Активная панель** (через `panel.handle_key()`)

**Поддержка кириллицы:**
Трансляция раскладки клавиатуры через `termide_keyboard::translate_hotkey()` позволяет горячим клавишам работать с русской раскладкой.

#### 3.3 Обработчик мыши

**Расположение:** `crates/app/src/app/mouse_handler.rs`

Обрабатывает ввод мыши:

**Заголовок панели:**
- Клик по кнопке `[≡]` → Открыть контекстное меню действий панели (Закрыть / Отделить / Объединить / Переместить)
- Клик по title → Активировать панель (двойной клик по файловому менеджеру → выбор каталога)
- **Drag за title** → Двухрежимный жест:
  - Drop внутри колонки исходной группы → вертикальный resize (делитель над панелью прыгает к курсору).
  - Drop в другой колонке или между колонками → перенос панели: ghost следует за курсором, drop-зона подсвечивается; drop на заголовок другой панели вставляет в её группу, drop между группами создаёт новую. `Escape` отменяет.

**Содержимое панели:**
- Клики перенаправляются в `panel.handle_mouse()`
- Каждая панель обрабатывает свои собственные взаимодействия с мышью

**Строка меню:**
- Клик по пунктам меню для активации

#### 3.4 Обработчик модальных окон

**Расположение:** `crates/app/src/app/modal_handler.rs` и `crates/modal/src/`

Обрабатывает интерактивные модальные диалоги:

**Типы модальных окон** (крейт `termide-modal`):
- **Input** — ввод текста (имя файла, имя директории и т.д.)
- **Confirm** — подтверждение Да/Нет
- **Select** / **EditableSelect** — выбор из вариантов (с возможным редактированием)
- **Choice** — горизонтальные кнопки выбора
- **Info** — отображение информации с **прокручиваемым содержимым** (отчёты скриптов, системная информация); скроллбар на правой рамке, клавиши `↑↓/PageUp/PageDown/Home/End` и колесо мыши
- **InfoAction** — info-окно с дополнительными кнопками действий
- **Settings** — полноэкранный модал конфигурации с **sidebar-layout**. Разбит на подмодули в `crates/modal/src/settings/`:
  - `settings.rs` — структура `SettingsModal`, рендер, обработка клавиш/мыши
  - `settings/fields.rs` — декларативные данные полей (`FieldType`, `FieldDescriptor`, `ContentRow`, функции `fields_for_tab`, `get_field_value`, `toggle_field`, `cycle_enum_*`)
  - `settings/kb.rs` — макросы и таблицы keybindings (`kb_get!`/`kb_set!`, `KB_SECTIONS`, `kb_binding_names`, `get/set_kb_value`, `format_key_event`)
- **Progress** — progress-bar для длительных операций
- **Commit** / **Conflict** / **RenamePattern** / **Search** / **Replace** / **TreeSearch** / **Sessions** / **DirectoryPicker** / **SaveAs** / **BookmarkAdd** / **Calendar** / **CommandPalette** / **ScriptCreate** — специализированные диалоги для отдельных операций

Общие утилиты вынесены в `crates/modal/src/base.rs` (`render_modal_block`, `render_modal_frame`, `button_style`, `CursorNavigation` trait).

**Захват ввода:**
Когда модальное окно открыто, ввод с клавиатуры сначала идёт в модальное окно. Escape закрывает модальное окно.

### 4. Конвейер рендеринга

#### 4.1 Основной рендеринг

**Расположение:** `crates/ui-render/src/layout.rs`

Поток рендеринга:

```rust
fn render_main_area(frame, layout_manager, state) {
    // 1. Вычислить ширины колонок (пропорциональные, с floor min_panel_width).
    let horizontal_chunks = calculate_horizontal_layout();

    // 2. Для каждой колонки взять вертикальные ограничения из кеша
    //    высот группы (или равномерное распределение по умолчанию).
    for group in groups {
        let vertical_chunks =
            termide_layout::compute_vertical_constraints(group, area_height);

        // 3. Отрисовать каждую панель. Панели с height >= 2 рисуются
        //    с полной рамкой и контентом; панели сжатые до одной строки
        //    рисуются как только заголовок.
        for (idx, panel) in group.panels().enumerate() {
            if vertical_chunks[idx].height >= 2 {
                let omit_bottom_border = idx + 1 < group.len();
                render_expanded_panel(panel, area, omit_bottom_border, ...);
            } else {
                render_collapsed_panel(panel, area, ...);
            }
        }
    }

    // 4. Отрисовать модальное окно (если открыто).
    if let Some(modal) = state.active_modal {
        render_modal(modal, ...);
    }
}
```

Каждая не-последняя панель не рисует свой нижний бордюр — заголовок следующей панели (с её title) служит разделителем, экономя строку на каждый стык. Угловые символы `┌`/`┐` следующей панели post-process'ятся в `├`/`┤`, чтобы псевдографика чисто соединялась с боковыми стенами `│` верхней панели.

#### 4.2 Рендеринг панелей

**Расположение:** `crates/ui-render/src/panel.rs`

**Полная панель (height ≥ 2 строк):**
- Граница с кнопкой действий `[≡]` и emoji + заголовком (напр. `[≡] 📁 Файлы`)
- Полная область содержимого
- Прокручиваемо, если содержимое превышает область
- Нижний бордер не рисуется, если ниже в той же группе есть ещё одна панель — её заголовок служит разделителем

**Сжатая панель (height = 1 строка, только когда сжата до минимума):**
- Только заголовок: `─[≡] 📁 Файлы ─────`
- Занимает одну строку
- Клик по заголовку фокусирует панель; `Alt+F11` или `Ctrl+Alt+=` увеличивает её

**Режим иконок:**
Заголовки панелей показывают emoji-иконки по типу панели (📁 файловый менеджер, 💻 терминал, 📝 редактор, 🔄 операции, 🚧 диагностика, 📑 outline, 🎨 image и т.д.). Все иконки выбраны так, чтобы надёжно занимать 2 клетки в современных терминалах — это сохраняет выравнивание заголовка после иконки. Режим настраивается через `icon_mode` в секции `[general]`:
- `auto` (по умолчанию) — emoji, если терминал поддерживает, иначе просто `[≡]`
- `emoji` — всегда показывать emoji-иконки
- `unicode` — без иконок, без стрелок, только `[≡]`

**Drag Overlay:**
При перетаскивании панели за верхний бордюр `render_drag_overlay()` (в `src/ui.rs`) вызывается после основного рендера панелей и до dropdown/modal. Он подсвечивает drop-target (верхний бордюр целевой панели для `IntoGroup`, вертикальную линию между группами для `NewGroup`) и рисует ghost-иконку под курсором. Хит-тест переиспользует free-функции `calculate_panel_rects` / `compute_drop_target` из `termide_layout`, так что mouse-обработчик и рендер согласованы по геометрии.

**Рендеринг границ:**
Границы и кнопки рисуются с помощью `panel_rendering.rs`, затем метод `render()` панели рисует содержимое во внутренней области.

### 5. Управление состоянием

#### 5.1 AppState

**Расположение:** `crates/state/src/` (разделён на `batch.rs`, `layout.rs`, `operations.rs`, `pending_action.rs`, `ui.rs`)

Центральный контейнер состояния:

```rust
pub struct AppState {
    pub theme: Theme,                    // Текущая тема
    pub terminal: TerminalInfo,          // Ширина, высота
    pub config: Config,                  // Конфигурация пользователя
    pub should_quit: bool,               // Флаг выхода
    pub batch_operation: Option<BatchOp>, // Ожидающие пакетные операции
    pub active_modal: Option<ActiveModal>, // Текущее модальное окно
    pub error_message: Option<String>,   // Ошибка для отображения
```

## Архитектурные решения

### Почему split-компоновка с fullscreen-пресетом?

**Проблема:** Пространство терминала ограничено, многопанельные IDE часто кажутся тесными — но и бинарная схема «одна развёрнутая, остальные свёрнуты» тоже отбрасывает случаи, когда пользователю нужно одновременно видеть две-три панели в подобранных пропорциях.

**Решение:** Регулируемый split per-group плюс однонажимный fullscreen-пресет:
- Все панели в группе видны по умолчанию; пользователь подбирает высоты мышью, `Ctrl+Alt+=` / `Ctrl+Alt+-` или просто перетаскиванием заголовка вверх/вниз внутри колонки.
- `Alt+F11` переключает «полноэкранный пресет» для активной панели — фокусная панель занимает всю колонку, остальные сжимаются до строки заголовка; высоты до пресета сохраняются для мгновенного restore.
- Высоты кешируются и пропорционально пересчитываются при ресайзе терминала.
- Автоматическое стекирование между группами по-прежнему срабатывает на узких терминалах (`min_panel_width`).

### Почему динамические панели?

**Преимущество:** Пользователи могут открывать столько панелей, сколько нужно:
- Несколько редакторов для разных файлов
- Несколько терминалов для разных задач
- Несколько файловых менеджеров для разных директорий

**Вызов:** Эффективное управление множеством панелей
- Fullscreen-пресет даёт чистый вид «одна панель на весь экран» по требованию
- Горячие клавиши обеспечивают быструю навигацию
- Экран приветствия автоматически закрывается

### Почему панели на основе трейтов?

**Гибкость:** Новые типы панелей можно добавлять без изменения основного кода
- Реализовать трейт `Panel`
- Добавить в логику создания панелей
- Работает с существующей системой компоновки

**Полиморфизм:** `Box<dyn Panel>` позволяет гетерогенные коллекции
- Единственный `Vec<Box<dyn Panel>>` содержит все типы панелей
- Единообразный рендеринг и обработка событий
- Накладные расходы динамической диспетчеризации незначительны для TUI

## Характеристики производительности

**Рендеринг:** O(n), где n = количество панелей в видимых группах
- Панели с height ≥ 2 рендерят полное содержимое; панели сжатые до 1 строки рендерят только заголовок
- Нижние бордюры между соседними панелями объединяются с заголовком следующей — экономит строку на каждый стык

**Обработка событий:** O(1) для большинства операций
- Прямой доступ по индексу к панели с фокусом
- Поиск в хеш-таблице для привязок клавиш

**Память:** Линейно от количества панелей
- Каждая панель владеет своим состоянием
- Общий AppState небольшой
- Нет избыточного клонирования (используются ссылки)

**Файловые операции:** Асинхронно, где возможно
- Наблюдатель файловой системы использует отдельный поток
- Дебаунсинг предотвращает избыточные обновления

### 8. Управление сессиями

**Расположение:** `crates/session/src/lib.rs`

Сохранение сессий позволяет сохранять и восстанавливать компоновку панелей:

**Расположение хранилища:**
- Linux: `~/.local/share/termide/sessions/<путь_проекта>/session.toml`
- macOS: `~/Library/Application Support/termide/sessions/<путь_проекта>/session.toml`

**Возможности:**
- Автоматическое сохранение сессии при выходе
- Восстановление компоновки панелей при запуске
- Переключение между сессиями через меню (переключение между разными проектами)
- Автоматическая очистка старых сессий

**Формат файла сессии:**
```toml
focused_group = 0

[[panel_groups]]
expanded_index = 1            # фокусная панель в группе
width = 80                    # ширина колонки (None = авто-распределение)
split_heights = [12, 6]       # per-panel высоты (отсутствуют, если кеша нет)
fullscreen_cache = [10, 8]    # высоты для restore при выключении Alt+F11

[[panel_groups.panels]]
type = "file_manager"
path_or_url = "/home/user/project"

[[panel_groups.panels]]
type = "editor"
path = "/home/user/project/main.rs"
```

Поле `mode = "accordion"` из старых сессий по-прежнему читается и инициирует одноразовую миграцию в fullscreen-пресет (текущий код его не пишет).

## Соображения безопасности

**Инъекция в терминал:**
- ANSI escape-последовательности фильтруются в терминальной панели
- Пользовательский ввод санитизируется перед выполнением в оболочке

**Файловые операции:**
- Атаки через симлинки предотвращаются
- Проверки path traversal
- Проверка прав доступа перед операциями

**Лимиты ресурсов:**
- Лимит размера файла (100 МБ) для редактора
- Лимит буфера прокрутки для терминала
- Ротация логов для предотвращения исчерпания диска

## Заключение

Архитектура TermIDE приоритизирует:
- **Гибкость** - Динамическая система панелей адаптируется к потребностям пользователя
- **Эффективность** - Регулируемая split-компоновка с однонажимным fullscreen-пресетом максимизирует используемое пространство, не отказываясь от многопанельных видов
- **Расширяемость** - Дизайн на основе трейтов позволяет лёгкие дополнения
- **Надёжность** - Оборонительное программирование предотвращает сбои
- **Производительность** - Эффективный рендеринг и обработка событий

Split-компоновка с fullscreen-пресетом — ключевая инновация, которая отличает TermIDE от традиционных многопанельных терминальных приложений.
