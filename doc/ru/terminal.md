# Терминал

Панель терминала предоставляет полноценный эмулятор терминала с поддержкой псевдотерминала (PTY), что обеспечивает совместимость с большинством консольных приложений, таких как `bash`, `fish`, `htop` и `mc`.

## Основные возможности

- **Интерактивная оболочка**: Запускает системную оболочку по умолчанию (`fish`, `zsh`, `bash` и т.д.) для выполнения команд.
- **Совместимость**: Поддерживает `xterm-256color` и большинство стандартных управляющих последовательностей ANSI, что обеспечивает корректное отображение цветов и стилей текста.
- **Совместимость с современными TUI**: Отвечает на типичные запросы о возможностях терминала и поддерживает согласованное keyboard/focus reporting, которое используют `vim`, `neovim`, `yazi`, `htop` и `lazygit`.
- **Управление процессами**: При закрытии панели терминала, если в ней запущены процессы, приложение запросит подтверждение на их завершение.

## Взаимодействие

| Комбинация             | Действие                                      |
|------------------------|-----------------------------------------------|
| `Ctrl+/`               | Открыть переключатель каталогов               |
| `Ctrl+F`               | Открыть поиск по тексту в скроллбэке          |
| `Ctrl+Shift+C`         | Копировать выделенный текст в буфер обмена    |
| `Ctrl+Shift+V`         | Вставить текст из буфера обмена               |
| `Ctrl+Shift+М`         | Вставить текст из буфера обмена (кириллица)   |
| `Shift+Enter`          | Вставить перенос строки (многострочный ввод)  |
| `Shift+PageUp`         | Прокрутить историю вывода вверх               |
| `Shift+PageDown`       | Прокрутить историю вывода вниз                |
| `Shift+Home`           | Перейти в начало истории вывода               |
| `Shift+End`            | Перейти к текущей строке (конец истории)      |

**Поддержка раскладки клавиатуры:**

TermIDE поддерживает кириллические раскладки клавиатуры для общих горячих клавиш. При использовании русской/кириллической раскладки вы можете использовать `Ctrl+Shift+М` (где М — это кириллическая буква, соответствующая V) вместо переключения на латинскую раскладку. Это работает для операций вставки в терминале.

Все остальные комбинации клавиш передаются напрямую в запущенное в терминале приложение.

Если приложение внутри терминала запрашивает современный режим отчётности клавиатуры, TermIDE переключается с legacy xterm-кодирования клавиш на согласованный режим совместимости. Это помогает приложению различать неоднозначные сочетания вроде `Ctrl+I` и `Tab`, `Ctrl+M` и `Enter`, а также модифицированные `Esc`/`Backspace`.

**Модифицированные стрелки и Home/End** кодируются как стандартная xterm
CSI-последовательность `1;{mod}{final}` (`{mod}` — параметр модификатора
xterm: `2` — Shift, `3` — Alt, `5` — Ctrl, `6` — Ctrl+Shift и т.д.;
`{final}` — `A`/`B`/`C`/`D`/`H`/`F`). На практике это означает, что
`Ctrl+Left` / `Ctrl+Right` вызывают `backward-word` / `forward-word` в
readline bash/zsh, `Shift+Home` / `Shift+End` выделяют до границ строки
там, где это поддерживает оболочка, и т.п. Немодифицированные стрелки
сохраняют прежний путь, включая подмену в application-cursor-mode
(`\x1bOA` vs `\x1b[A`). `Alt+Left` / `Alt+Right` по-прежнему зарезервированы
глобальной раскладкой (переключение предыдущая/следующая группа панелей)
и не пробрасываются в терминал.

## Поиск по тексту

Нажмите `Ctrl+F`, чтобы открыть встроенную строку поиска, докнутую сверху панели
(тот же UX, что в редакторе и файловом менеджере), с разделительной линией под
ней. Поиск работает по всему скроллбэку и видимому экрану:

- **Живой предпросмотр**: совпадения подсвечиваются по мере ввода; в строке
  отображается счётчик совпадений (например, "3 из 12")
- **Тумблеры**: `[Aa]` учёт регистра и `[.*]` регулярные выражения (кликом или
  фокусом на ряду кнопок и нажатием `Enter` / `Space`)
- **Навигация**: кнопки `◄ Prev` / `Next ►`, `Enter` или `F3` / `Shift+F3`
  переходят между совпадениями; область просмотра прокручивается к текущему
- **Фокус**: `Tab` переключает фокус между строкой и сеткой терминала — сетку
  можно прокручивать, пока строка открыта
- **Закрытие**: `Escape`

По умолчанию привязка поиска — `Ctrl+F` (а не `Ctrl+Shift+F`), так как большинство хост-терминалов перехватывают `Ctrl+Shift+F` для собственного поиска.

## Выбор оболочки

Выбрать оболочку для запуска можно через подменю **Windows > Terminal**. В подменю отображаются все обнаруженные в системе оболочки:

- **Linux/macOS**: оболочки из `/etc/shells`, а также типичные пути (`/usr/bin/fish`, `/usr/bin/zsh`, `/bin/bash`, `/bin/sh`) и пути NixOS
- **Windows**: Git Bash, PowerShell Core (`pwsh`), Windows PowerShell, командная строка (`cmd`) и дистрибутивы WSL

Текущая оболочка по умолчанию отмечена символом **●**. При выборе оболочки открывается новый терминал с этой оболочкой, и она сохраняется как оболочка по умолчанию для будущих терминалов.

Также можно задать оболочку по умолчанию в `config.toml`:

```toml
[terminal]
default_shell = "/usr/bin/fish"
```

## Поддержка мыши

- **Выделение текста**: В обычной оболочке и скроллбэке используйте левую кнопку мыши и перетаскивание. Если приложение внутри терминала включает xterm mouse tracking, для локального выделения текста используйте `Alt+drag`.
- **Колесо прокрутки**: Прокрутка истории вывода терминала до тех пор, пока приложение внутри терминала не включит mouse tracking. После этого колесо передаётся приложению.
- **Ctrl+Клик на URL/путь**: Открыть ссылку в браузере или файловом менеджере.
- **Ctrl+Клик на hex-цвет**: Показать превью цвета (например, `#ff0000`, `#abc`) — отображается пока кнопка зажата, исчезает при отпускании.
- **Взаимодействие с приложениями**: Если консольное приложение (например, `htop` или `mc`) включает xterm mouse tracking, TermIDE отдаёт ему приоритет для кликов, перетаскивания, движения курсора и колеса мыши внутри области терминала.

## Примечания по совместимости приложений

- **Согласование клавиатуры**: TermIDE отвечает на распространённые запросы о поддержке клавиатурных расширений и по запросу приложения включает совместимый режим `CSI u` / `modifyOtherKeys`.
- **Отчёты о фокусе**: Если приложение включает xterm focus events (`?1004`), TermIDE передаёт получение и потерю фокуса хост-терминала как `CSI I` / `CSI O`.
- **Идентичность терминала**: Внутренний PTY сохраняет обычную базу `xterm-256color`, а дополнительные возможности даёт через runtime negotiation, а не через подмену на другой тип терминала.
