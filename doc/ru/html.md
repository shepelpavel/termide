# Просмотр HTML

TermIDE отображает файлы HTML (`.html`, `.htm`) как панель рендеренного
просмотра (только чтение) — текстовая псевдографика, а не сырой исходник. Это
*не* браузер: авторский CSS и скрипты игнорируются, а раскладка — это
фиксированное отображение «тег → стиль».

## Открытие

- **`F3`** на `.html`-файле — открыть рендеренный просмотр.
- **`Enter`** или **`F4`** — открыть сырой исходник в редакторе (как для любого
  текстового файла).

## Просмотр ↔ Исходник

Просмотр и редактор исходника — это два представления одного файла. Переключение
происходит на месте: панель заменяется, а не складывается в стопку:

- **`Ctrl+E`** (настраивается через `[viewer.keybindings] toggle_view`).
- Кликабельный чип **`Edit`** в статус-баре.

В просмотре `Edit: No` означает, что вы смотрите рендеренный документ; клик по
нему (или `Ctrl+E`) открывает **редактируемый** исходник. В редакторе исходника
тот же переключатель возвращает к просмотру. Возврат к просмотру блокируется,
пока в исходнике есть несохранённые изменения — сначала сохраните.

## Что рендерится

Токенизация через `html5ever` (без DOM, без CSS), вывод в виде текстовой
псевдографики. Поддерживаемое подмножество тегов отображается в стилизованные
блоки и инлайновые фрагменты; **неизвестные теги прозрачны** — их содержимое
всё равно рендерится. HTML-сущности (`&amp;`, `&#39;`) декодируются; содержимое
`<script>`, `<style>` и `<head>` отбрасывается.

- Заголовки `<h1>`–`<h6>` (с префиксами `#`, акцентный цвет, жирный).
- Параграфы и блочные контейнеры (`<p>`, `<div>`, `<section>`, …), разделённые
  пустыми строками.
- Инлайновое выделение: `<b>`/`<strong>`, `<i>`/`<em>`, `<u>`, `<s>`/`<del>` и
  `<kbd>`/`<mark>` (инверсия).
- Инлайновый `<code>` и блоки `<pre>` (дословно).
- Маркированные `<ul>` и нумерованные `<ol>` списки; `<blockquote>` с префиксом
  `│`.
- `<table>` (с заголовочными строками `<thead>`), нарисованная псевдографическими
  рамками.
- Ссылки `<a href>` (подчёркнуты, кликабельны) и `<img>` — кликабельная
  пиктограмма `🖼` с альт-текстом.
- Переводы строк `<br>`, линии `<hr>` и `<details>`/`<summary>` (показывается
  раскрытым).

Тот же движок отрисовывает HTML, встроенный в [просмотр Markdown](markdown.md).

## Навигация, выделение, ссылки

У просмотра есть подвижный курсор и поддержка выделения текста:

- `↑`/`↓`/`←`/`→` (или `k`/`j`/`h`/`l`) — двигать курсор.
- `PageUp`/`PageDown` (или `Space`) — страница вверх/вниз; `Home`/`End` — концы
  строки; `g`/`G` — в начало/конец документа.
- С зажатым **`Shift`** при движении или **перетаскиванием мышью** — выделение
  текста.
- **`Ctrl+C`** копирует выделение (или строку под курсором, если выделения нет)
  в буфер обмена.
- **`Ctrl+F`** открывает инкрементальный поиск; **`Ctrl+R`** перечитывает файл
  с диска.
- **`Ctrl+G`** запрашивает путь **или `http(s)://` URL** и открывает его в
  подходящем просмотрщике (HTML, Markdown, изображение или текст) — быстрый
  переход к соседнему файлу или простой текстовый просмотр веб-страницы (см.
  *Загрузка URL* ниже).
- Колесо мыши прокручивает.
- **Переход по ссылке** — клик или `Enter`, когда курсор на ней. По умолчанию
  ссылки открываются **в панели**: загруженная (URL-backed) страница навигирует
  на месте (относительные ссылки резолвятся относительно URL страницы),
  веб-ссылка из файлового просмотра открывается в новой панели, а ссылка на
  **изображение** — в просмотрщике картинок. Место открытия по умолчанию задают
  две настройки — `[viewer] open_links` для страниц и `[viewer] open_images`
  для ссылок на изображения — каждая `panel` (по умолчанию) или `external`.
- **`O`** всегда открывает ссылку под курсором во внешнем браузере (независимо
  от настройки).
- **`[` / `]`** (или **`Backspace`** для «назад») — шаг назад/вперёд по истории
  навигированного просмотра.
- **Якорные ссылки** (`#section`) прыгают внутри страницы (к элементу с этим
  `id`); `page#section` переходит на страницу и затем прыгает к якорю.

## Загрузка URL

Пункт меню **Windows ▸ Web** открывает запрос URL напрямую (обнаружимая точка
входа); `Ctrl+G` в любом просмотрщике делает то же. `Ctrl+G` с адресом
`http(s)://` загружает документ в фоне и открывает его по
`Content-Type` (HTML → этот просмотрщик, Markdown → просмотрщик Markdown,
прочий текст → дословно). Загрузка намеренно ограничена — это читалка, а не
движок браузера:

- только `http` и `https`; TLS проверяется (без отключения).
- таймаут 15 секунд; не более 5 редиректов, `https` никогда не понижается до
  `http`.
- ответы больше 8 МиБ отклоняются.
- встроенные ресурсы **не** загружаются — `<img>` остаётся пиктограммой `🖼`,
  поэтому страница не может «позвонить домой» через просмотрщик.

Ссылки внутри загруженной страницы открываются **на месте** (`Enter`/клик),
относительные ссылки резолвятся относительно URL страницы, история — по
`[`/`]` (или `Backspace`); `O` открывает ссылку в настоящем браузере. Виды,
загруженные по URL, не восстанавливаются между сессиями.

Файловая панель сохраняется между сессиями и открывается на том же файле.
