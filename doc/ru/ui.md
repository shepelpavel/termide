# Общее устройство окна приложения

**Приложение занимает всё доступное место и имеет вертикальное разделение на элементы:**
- Строка меню
- Область панелей инструментов
- Статусная строка

Также приложение использует всплывающие окна:
- Помощь
- Настройки
- Подтверждение закрытия приложения

## Модальные окна

Приложение использует интерактивные модальные окна для различных операций:
- **Встроенные строки поиска / замены** (`Ctrl+F` / `Ctrl+H`) - Докнутые поиск и замена прямо в редакторе, файловом менеджере и терминале: живой предпросмотр, счётчик совпадений, тумблеры `[Aa]` / `[.*]` и кнопки навигации
- **Модалы ввода** - Различные запросы для файловых операций (создание, переименование и т.д.)
- **Диалоги подтверждения** - Подтверждение удаления, несохранённых изменений и т.д.

**Возможности модалов:**
- Кнопка `[X]` для закрытия в заголовке модала (нажимается мышью)
- Навигация с клавиатуры через Tab/Shift+Tab (между полями / кнопками модала)
- Поддержка мыши для всех кнопок
- Клавиша Escape для закрытия модала
- Живой предпросмотр для операций поиска/замены
- Сохранение состояния (последний введённый текст сохраняется)

### Модал настроек (Settings)

Открывается клавишей `Alt+P` или через меню **Options → Edit preferences**. В отличие от прежнего горизонтально-табового варианта, теперь используется **sidebar-layout**:

**Три зоны:**
- **Левый sidebar** (ширина 18 символов) — список секций с навигацией Up/Down
- **Область контента** — поля выбранной секции, сгруппированные подзаголовками `── Группа ──` и разделённые пустыми строками
- **Кнопки снизу** — `Apply & Save`, `Reset`, `Cancel`

**Секции в sidebar:**
- General, Editor, File Manager, Terminal, LSP, Logging, VFS — обычные листья
- **Keybindings** — раскрываемая группа (▶/▼) с 7 подсекциями (Global, Editor, FileManager, GitStatus, GitDiff, GitLog, Terminal)

**Навигация:**

| Комбинация | Действие |
|-----------|----------|
| `Up` / `Down` | Перемещение внутри текущей зоны (секции / поля / кнопки) |
| `Tab` / `Shift+Tab` | Переключение между зонами (Sidebar ↔ Content ↔ Buttons) |
| `Left` / `Right` | **Не** меняют зону: используются для изменения значений (cycle enum, toggle bool) и для свёртки группы Keybindings в sidebar |
| `Enter` / `Space` | Активировать: в sidebar — войти в секцию, в content — переключить bool/enum или начать редактирование текста/числа, на header группы — toggle expand |
| `Escape` | Закрыть модал (Cancel) |
| Колесо мыши | Прокрутка sidebar / content |

**Поля и индикаторы:**
- **Bool** — `[✓]` (включено) или `[✗]` (выключено), переключаются `Enter`/`Space` или `Left`/`Right`
- **Enum** — `< значение >`, циклятся `Left`/`Right`
- **Number** / **OptionalText** — `Enter` входит в режим inline-редактирования
- **LSP → Servers** — список серверов помечен буллетом `•`; есть строка `+ Add Server`

**Keybindings — захват клавиши:**  
Навигация `Up/Down` в списке бинда, `Enter` — режим Capturing (далее любая нажатая клавиша становится новым биндом), `Delete`/`Backspace` — очистить бинд.

Подсветка активной секции в sidebar убирается при потере фокуса — текущая секция видна по заголовку в области контента, поэтому двойная подсветка не вводит в заблуждение.

## Строка меню

Строка меню располагается вверху окна и включает: пункты меню расположенные слева, индикаторы системных ресурсов (CPU, RAM, скорость сети) и часы в формате ЧЧ:ММ расположенные справа.
Активация и деактивация меню, а также каждого пункта может производиться кликом мыши либо [клавиатурной комбинацией](#Клавиатурная-навигация-и-управление-по-меню-и-панелям).

**Пункты меню:**
- `Sessions` — управление сессиями:
  - Новая сессия — создать сессию в новой директории
  - Переключить сессию — открыть модальное окно переключения
  - Изменить корневой путь — переместить текущую сессию в другую директорию
- `Windows` — создание панелей:
  - Files — файловый менеджер
  - Terminal — терминал (имеет подменю выбора оболочки: показывает все доступные в системе оболочки, текущая оболочка по умолчанию отмечена ●)
  - Editor — текстовый редактор
  - Git Status — панель статуса git
  - Git Log — история коммитов
  - Git Stash — панель управления стэшами
  - Journal — журнал приложения
  - Diagnostics — диагностика LSP
  - [Operations](operations.md) — фоновые операции
  - Outline — структурная навигация по коду
- `Scripts` — пользовательские скрипты (с группами подменю). Клик по заголовку группы разворачивает подменю; повторный клик на том же заголовке — сворачивает (toggle).
- `Bookmarks` — закладки (директории, файлы, SSH, SFTP). Поведение групп — toggle, как и в Scripts.
- `Options` — настройки:
  - Themes — выбор темы с предпросмотром
  - Language — язык интерфейса с предпросмотром
  - Manage scripts — открыть папку скриптов
  - Manage bookmarks — открыть файл закладок
  - Edit preferences — открыть config.toml в редакторе
  - Help — панель помощи
  - Quit — выход

**Индикаторы системных ресурсов** (все кликабельны — открывают модальное окно с подробностями):

| Индикатор | Описание | Клик открывает |
|-----------|----------|----------------|
| `CPU XX%` | Загрузка процессора | Топ-10 процессов по CPU |
| `RAM X/YГб` | Использование памяти | Топ-10 процессов по RAM |
| `↓…/↑…` | Скорость сети (↓ загрузка, ↑ отдача) | Топ-10 процессов по сетевой активности |
| `УСТРОЙСТВО занято/всегоГб` (в статусной строке) | Использование диска | Топ-10 процессов по диску + разделы ФС |

Повторный клик по тому же индикатору закрывает открытое им окно (toggle), то же поведение и у индикатора диска в статусной строке.

Цветовая кодировка индикаторов: зелёный < 50%, жёлтый 50–75%, красный > 75%.

**Модал сетевой активности** показывает:
- **Application** — имя процесса
- **Ports** — слушающие TCP-порты (например, `80, 443`)
- **Conn** — количество активных (ESTABLISHED) соединений

Процессы отсортированы по убыванию числа соединений.

## Область панелей инструментов

Область заполняет вертикальное пространство между строкой меню и статусной строкой от левого до правого края окна.
Раскладка адаптируется к ширине терминала, показывая больше групп панелей на широких экранах. Панели (терминал, редактор, git) могут быть открыты и будут размещены рядом.

Новая панель всегда открывается **после текущей активной** в аккордеоне группы (не в конце списка). При закрытии панели фокус переходит на предыдущую.

**Возможные варианты открываемых панелей:**
- [файловый менеджер](file-manager.md) — `Alt+F`
- [терминал](terminal.md) — `Alt+T`
- [текстовый редактор](editor.md) — `Alt+E`
- статус git — `Alt+G`
- структура (outline) — `Alt+O`
- диагностика — `Alt+I`
- git лог — `Alt+C`
- git diff
- [операции](operations.md)
- просмотр изображений
- помощь — `Alt+H`
- журнал — `Alt+L`

Панель **git лог** рисует граф коммитов псевдографикой (`● │ ├ ╮ ╯`),
раскладывая его по родителям каждого коммита; каждая дорожка раскрашена своим
цветом, чтобы ветку можно было вести глазом по цвету. Чтобы вернуться к нативному
ASCII-выводу git `--graph`, задайте в `config.toml`:

```toml
[git_log]
unicode_graph = false
```

Открытие **бинарного файла** (по `Enter` или `F3` в файловом менеджере)
показывает read-only **hex-вьюер** — `смещение │ hex │ ASCII`, ширина строки
подстраивается под панель секциями по 16 байт — вместо передачи системному
просмотрщику. Курсор байта виден сразу в hex- и ASCII-зонах (`Tab` переключает
активную); стрелки/`hjkl` двигают его, `Shift`+движение выделяет, `Ctrl+C`
копирует выделение (как hex из hex-зоны, как текст из ASCII-зоны). `Ctrl+F`
открывает поиск по ASCII-подстроке или — с тумблером `[hex]` — по
последовательности байт вида `ff fe`.

`Ctrl+L` (или чип `[Hex]` в статус-баре) подменяет вьюер на месте read-only
текстовым редактором того же файла; `Ctrl+L` в редакторе возвращает в hex.
Клавиша настраивается:

```toml
[viewer.keybindings]
toggle_hex = "Ctrl+L"
```

**Особенности закрываемых панелей:**
- Имеют кнопку `[≡]` в заголовке панели (клик открывает контекстное меню с пунктами Закрыть / Отделить / Объединить / Переместить)
- Могут быть закрыты клавишами Escape, Alt+X или F10
- Ширина колонки регулируется клавишами `Alt+=` / `Alt+-`
- Высота панели внутри колонки регулируется `Alt+Shift+=` / `Alt+Shift+-` (шаг 1 строка), перетаскиванием нижнего бордюра панели мышью или перетаскиванием заголовка панели вверх/вниз внутри колонки
- `Alt+F11` переключает «полноэкранный пресет» для активной панели (одна панель занимает всю колонку, остальные сжимаются до строки заголовка); повторное нажатие восстанавливает прежние высоты
- Могут перетаскиваться за верхний бордюр (см. раздел «Взаимодействие мышью» ниже)

**Контекстное меню действий панели:**

Клик по кнопке `[≡]` в заголовке панели (или нажатие `Alt+K` / `Shift+F10` на активной панели) открывает выпадающее меню:
- **Переместить вверх / Переместить вниз** — переупорядочить внутри текущей группы (если в группе больше одной панели)
- **Переместить влево / Переместить вправо** — в соседнюю группу (если групп больше одной)
- **Отделить / Объединить** — отделить панель из группы в новую (если в группе больше одной панели) или объединить одиночную панель с соседней группой
- **Закрыть** — закрыть панель (с подтверждением при необходимости)

Пункты фильтруются по контексту: например, *Отделить/Объединить* скрывается, если всего одна панель в одной группе; *Переместить вверх/вниз* — для одиночной панели в группе.

**Взаимодействие мышью:**
- Клик по заголовку активирует панель; двойной клик по заголовку
  файлового менеджера открывает выбор каталога.
- **Перетаскивание нижнего бордюра панели** в колонке — изменение
  высоты этой панели (живая ghost-линия `━` показывает положение
  нового разделителя, ресайз применяется на отпускании).
- **Перетаскивание панели за верхний бордюр (title)** — режим
  resize или reorder:
  - Отпускание в теле исходной панели или в теле верхнего соседа
    → вертикальный resize: разделитель над панелью прыгает к
    курсору (у верхней панели колонки разделителя выше нет —
    жест переходит в режим переноса).
  - Отпускание на строке-заголовке другой панели в той же группе
    → переупорядочивание: вставить перед / после в зависимости
    от направления.
  - Отпускание над телом панели другой колонки → перенос в эту
    группу с разрезанием целевой панели на строке drop'а.
  - Отпускание в 2-клеточном зазоре между группами → новая группа
    в этом месте.
  - Отпускание правее последней группы → добавление как новой
    последней группы.
  - Во время drag жирная линия `━` отмечает горизонтальные
    drop-зоны (resize / split в колонке), жирная `┃` —
    вертикальные (новая колонка между группами).
  - Нажатие `Escape` отменяет drag.

## Статусная строка

Статусная строка предназначена для отображения дополнительных сведений о работе в активной панели.
В зависимости от типа активной панели в ней отображаются соответствующие данные.

### Индикатор дискового пространства

Справа в статусной строке отображается информация о дисковом пространстве в формате: `УСТРОЙСТВО занято/всегоГб (процент%)` с цветовой кодировкой в зависимости от уровня заполненности:

**Цветовая кодировка:**
- **Зелёный** при заполненности диска < 50%
- **Жёлтый** при заполненности диска 50-75%
- **Красный** при заполненности диска > 75%

**Формат:** `УСТРОЙСТВО занято/всего (процент%)`

Пример: `NVME0N1P2 386/467Гб (83%)`

Название устройства определяется автоматически из файловой системы:
- В Linux: показывает имена разделов как `NVME0N1P2`, `SDA1` и т.д.
- В macOS: показывает идентификаторы дисков
- Отображаемое устройство соответствует разделу, на котором находится текущая директория

### Caps Lock

Termide включает [Kitty keyboard protocol](https://sw.kovidgoyal.net/kitty/keyboard-protocol/) через `KeyboardEnhancementFlags::DISAMBIGUATE_ESCAPE_CODES | REPORT_ALTERNATE_KEYS | REPORT_EVENT_TYPES`. `REPORT_EVENT_TYPES` пробрасывает `KeyEventState::CAPS_LOCK`, и матчер хоткеев использует этот бит, чтобы игнорировать «фантомный» `Shift`, который X11/Linux-терминалы навешивают на каждый буквенный event при включённом Caps Lock.

Что это даёт на практике:

- Бинты вроде `Alt+T` продолжают работать при включённом Caps Lock (без фикса event пришёл бы как `{Char('T'), Alt|Shift}` и не совпал бы с `{Char('t'), Alt}`).
- Намеренные `Shift+буква`-бинты (например, `Ctrl+Shift+F` для content-поиска) остаются отличимыми от безшифтовых версий — Shift игнорируется только если реально сообщён Caps Lock.
- Терминалы без поддержки Kitty protocol (бит Caps Lock не приходит) падают на строгое сравнение модификаторов, поведение там прежнее — лучше использовать такие терминалы с выключенным Caps Lock.

Про кодировку модифицированных стрелок и Home/End для встроенного эмулятора терминала см. [terminal.md](terminal.md#взаимодействие).
