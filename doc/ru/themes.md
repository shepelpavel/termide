## Переключение тем

### Метод 1: Через меню (Рекомендуется)

1. Нажмите на **Options** в строке меню
2. Выберите **Themes** в выпадающем списке
3. Выберите желаемую тему из списка - каждый пункт показывает превью цветов темы
4. Тема применится немедленно и сохранится в конфигурации

### Метод 2: Через горячую клавишу

1. Нажмите `Alt+P` во время работы TermIDE чтобы открыть файл конфигурации во встроенном редакторе
2. Найдите параметр `theme`
3. Измените его на желаемое название темы (например, `theme = "dracula"`)
4. Сохраните файл нажав `Ctrl+S` - новая тема применится немедленно

### Метод 3: Ручное редактирование

Вы также можете отредактировать файл конфигурации напрямую любым текстовым редактором:

**Linux:**
```bash
~/.config/termide/config.toml
```

**macOS:**
```bash
~/Library/Application Support/termide/config.toml
```

**Windows (WSL):**
```bash
~/.config/termide/config.toml
```

Измените параметр `theme`:
```toml
theme = "dracula"
language = "auto"
```

Если вы редактируете файл во время работы TermIDE и он открыт через `Alt+P`, тема применится немедленно при сохранении. В противном случае, новая тема применится при следующем запуске TermIDE.

## Пользовательские темы

Вы можете создавать собственные темы, размещая TOML файлы в директории тем.

### Расположение директории тем

**Linux:**
```bash
~/.config/termide/themes/
```

**macOS:**
```bash
~/Library/Application Support/termide/themes/
```

**Windows (WSL):**
```bash
~/.config/termide/themes/
```

### Создание пользовательской темы

1. Создайте новый `.toml` файл в директории тем:
   ```bash
   mkdir -p ~/.config/termide/themes
   nano ~/.config/termide/themes/my-theme.toml
   ```

2. Определите цвета темы используя структуру ниже.

3. Установите вашу тему в конфигурации:
   ```toml
   theme = "my-theme"
   ```

Пользовательские темы имеют приоритет над встроенными темами с тем же именем.

## Структура файла темы

Файл темы — это TOML файл со следующей структурой:

```toml
# Метаданные темы
name = "my-theme"

[colors]
# Базовые цвета
bg = { rgb = [20, 20, 20] }          # Цвет фона (RGB)
fg = "White"                          # Цвет текста

# Акцентированные элементы (активная панель, фокусированные элементы)
accented_bg = { rgb = [40, 40, 40] }  # Акцентированный фон
accented_fg = "Green"                 # Акцентированный текст

# Выделение (выбранные файлы, выделение текста)
selected_bg = "Blue"                  # Фон выделения
selected_fg = "White"                 # Текст выделения

# Неактивные элементы интерфейса
disabled = "Gray"                     # Цвет неактивных элементов

# Семантические цвета (индикаторы состояния)
success = "Green"                     # Сообщения об успехе, высокие ресурсы
warning = "Yellow"                    # Предупреждения, средние ресурсы
error = "Red"                         # Ошибки, низкие ресурсы
```

### Форматы цветов

Цвета могут быть указаны двумя способами:

**1. Именованные цвета:**
```toml
fg = "White"
bg = "Black"
error = "Red"
```

Поддерживаемые именованные цвета: `Black`, `Red`, `Green`, `Yellow`, `Blue`, `Magenta`, `Cyan`, `White`, `Gray`, `DarkGray`

**2. Значения RGB:**
```toml
bg = { rgb = [20, 20, 20] }
accented_bg = { rgb = [40, 40, 40] }
selected_bg = { rgb = [0, 120, 215] }
```

Значения RGB варьируются от 0 до 255 для каждого канала (красный, зелёный, синий).

## Использование цветов темы

Различные цвета используются для разных элементов интерфейса:

| Цвет | Используется для |
|------|------------------|
| `bg` / `fg` | Стандартный фон и текст |
| `accented_bg` / `accented_fg` | Рамки активной панели, фокусированные элементы |
| `selected_bg` / `selected_fg` | Выбранные файлы, выделение текста в редакторе |
| `disabled` | Неактивные элементы UI, серый текст |
| `success` | CPU/RAM/Диск < 50%, успешные сообщения |
| `warning` | CPU/RAM/Диск 50-75%, предупреждения |
| `error` | CPU/RAM/Диск > 75%, сообщения об ошибках |

## Пример: Создание пользовательской тёмной темы

```toml
name = "my-dark-theme"

[colors]
# Тёмный фон со светлым текстом
bg = { rgb = [30, 30, 30] }
fg = { rgb = [220, 220, 220] }

# Фиолетовые акценты
accented_bg = { rgb = [60, 40, 80] }
accented_fg = { rgb = [200, 150, 255] }

# Голубое выделение
selected_bg = { rgb = [0, 150, 200] }
selected_fg = "White"

# Серый для неактивных элементов
disabled = { rgb = [100, 100, 100] }

# Стандартные семантические цвета
success = "Green"
warning = "Yellow"
error = "Red"
```

Сохраните это как `~/.config/termide/themes/my-dark-theme.toml` и установите `theme = "my-dark-theme"` в конфигурации.
