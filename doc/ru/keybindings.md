# Клавиатурные сокращения

Termide канонизирует каждое событие клавиатуры перед сравнением с
бин­дом. Канонизация выполняется один раз на диспетчинге, и её
результат вместе с исходным сырым событием передаётся вниз по
конвейеру в виде `KeyChord`. Панели и модальные окна выбирают
нужную форму:

- **`canonical`** — для матчинга хоткеев, интерпретации vim-команд,
  захвата клавиш в Settings.
- **`raw`** — для ввода текста (`InsertChar`), проброса в PTY
  терминальной панели, ввода в поисковой строке.

Текст, набираемый в редакторе или отправляемый программе внутри
терминальной панели, **никогда** не переписывается канонизацией —
кириллица, shifted-знаки, locale-зависимые символы доходят до
адресата без изменений.

## Что исправляет канонизация

| Особенность | Поведение |
| --- | --- |
| Кириллические буквы на той же физической клавише, что и латиница (`й`/`q`, `ь`/`m`, …) | Маппятся в латиницу — бинд `Alt+M` срабатывает и при QWERTY, и при ЙЦУКЕН. |
| Перевод `REPORT_ALTERNATE_KEYS` shifted-glyph | Crossterm переписывает `Shift+Ctrl+=` → `Char('+') + Ctrl` (Shift сожран, символ заменён). Нормализатор обращает: `Char('+') + Ctrl` → `Char('=') + Ctrl + Shift`. |
| Caps Lock и ложный Shift на буквах | Когда `REPORT_EVENT_TYPES` пометил `KeyEventState::CAPS_LOCK`, флаг Shift на буквах сбрасывается перед сравнением. |
| Коллапс `Ctrl+/` в VTE → `Ctrl+7` | Только если Kitty proto **не** активен: `Ctrl+7` → `Ctrl+/`. |

## Универсальные vs расширенные биндинги

Некоторые хорды не могут быть закодированы любым терминалом. Termide
делит дефолты на два уровня и предупреждает при старте, если активный
терминал не может доставить настроенные хорды расширенного уровня.

### Универсальные (работают на любом VT100+ терминале)

- `Alt+буква`, `Ctrl+буква` (буквы → ASCII control 0x01–0x1A).
- `F1`–`F12` и `F1`–`F12` с **одним** модификатором (`Shift+F*`,
  `Alt+F*`, `Ctrl+F*`).
- Стрелки с **одним** модификатором (`Shift+Up`, `Ctrl+Up`, `Alt+Up`).
- `Home`, `End`, `PgUp`, `PgDn` + один модификатор.
- `Enter`, `Tab`, `Esc`, `Backspace`, `Delete`, `Insert` + один
  модификатор.
- `Alt+цифра`.
- `Alt+пунктуация` (`Alt+/`, `Alt+,`, `Alt+.`, …).

### Расширенные (требуют Kitty keyboard protocol)

- `Ctrl+пунктуация` (`Ctrl+/`, `Ctrl+-`, `Ctrl+=`, `Ctrl+,`, `Ctrl+.`).
- `Ctrl+Shift+буква`.
- `Ctrl+Alt+что-либо`.
- `Alt+Shift+буква` и `Alt+Shift+стрелка` — VTE в legacy режиме отдаёт
  `\eL` для `Alt+Shift+l`, неотличимое от `Alt+L`; биндинг
  `Alt+Shift+...` не сможет совпасть.
- Модификаторы `Super` / `Meta` / `Hyper`.

Дефолты расширенного уровня (`Ctrl+/` для `toggle_comment` и
`switch_directory`, `Ctrl+Alt+R` для `replace_all`) сохраняются как
де-факто стандарты редакторов. На терминале без Kitty proto termide
журналирует на старте предупреждение со списком затронутых биндов;
пользователь может перепривязать их через Settings → Keybindings.

## Совместимость терминалов (2026)

| Терминал | Kitty keyboard protocol |
| --- | --- |
| kitty | полная |
| foot 1.13+ | полная |
| WezTerm | полная |
| Ghostty | полная |
| iTerm2 | полная |
| rio | полная |
| Windows Terminal Preview 1.25+ | полная |
| alacritty | частичная (CSI-u, без enhancement flags) |
| xterm | частичная (требует ручной настройки) |
| GNOME Terminal / Tilix / VTE | нет (в разработке) |
| Konsole | нет (планируется) |
| tmux | проксирует (зависит от хоста) |

Если ваш терминал не объявляет Kitty proto и вы полагаетесь на хорды
расширенного уровня, либо переключитесь на поддерживающий терминал,
либо переопределите задействованные действия на универсальные хорды
в `config.toml` → `[*.keybindings]`.

## Обнаружение конфликтов

Settings → Keybindings показывает inline-предупреждение при
назначении хорда, уже используемого другим действием. Обнаруживаются
три класса конфликтов:

- **В одной секции** — два действия одной секции делят хорд; второе
  становится недостижимым.
- **Cross-section shadow** — глобальный хорд перекрывает панельный;
  панельный никогда не сработает.
- **Cross-section ambient** — пересечение между двумя панелями;
  только активная панель получит событие, обычно безопасно, но
  стоит знать.

Конфликты в одной секции также журналируются при старте.

## Переопределение дефолтов

Любой биндинг можно переопределить в `config.toml`. Строки парсятся в
канонической форме, поэтому `"Alt++"` ≡ `"Alt+Shift+="` и
`"Ctrl+Й"` ≡ `"Ctrl+Q"`:

```toml
[general.keybindings]
panel_grow_vertical = "Alt+Shift+="
panel_shrink_vertical = "Alt+Shift+-"
open_sessions = "Alt+\\"

[editor.keybindings]
trigger_completion = ["Ctrl+J", "Ctrl+Space"]
toggle_comment = ["Ctrl+/", "Ctrl+."]
replace_all = ["Ctrl+Alt+R", "Alt+R"]

[file_manager.keybindings]
switch_directory = "Ctrl+\\"

[terminal.keybindings]
switch_directory = "Ctrl+\\"
```

Примечание: `Ctrl+/` и `Ctrl+\` работают даже на legacy-терминалах
(например VTE) через quirks в `KeyNormalizer` — VTE шлёт их как
control-байты `\x1F` и `\x1C`, которые crossterm парсит в `Ctrl+7` /
`Ctrl+4`, а нормализатор переводит обратно в слэш / обратный слэш.

Для любого действия поддерживается несколько альтернатив — перечислите
их в массиве. Первая форма — канональная строка, отображаемая в
панелях помощи.
