# 数据库查看器

TermIDE 内置了一个**只读**数据库浏览器，支持 **SQLite**、**PostgreSQL** 和
**MySQL/MariaDB**。它通过原生协议连接（无需外部客户端工具），列出数据库的表，
并在可滚动的表格中显示行，支持排序和按列过滤。

查看器从不写入：只执行 `SELECT` 和目录查询。

## 打开连接

连接就是路径为数据库 URL 的普通**书签**。添加书签（书签菜单 → 添加），并在*路径*
字段中填入连接 URL：

| 引擎 | URL 示例 |
|------|----------|
| SQLite | `sqlite:///absolute/path/to/app.db` |
| PostgreSQL | `postgres://user@host:5432/mydb` |
| MySQL / MariaDB | `mysql://user@host:3306/mydb` |

激活数据库书签会打开数据库面板并在后台连接——连接时界面不会阻塞。

### 密码

与 SFTP/FTP 书签一样，密码不做特殊处理：书签存储的就是你输入的 URL。你可以依赖
免密码的认证方式（PostgreSQL 的 `~/.pgpass`/peer/trust、Unix 套接字、
`PGPASSWORD` 等），或将密码直接写入 URL（`postgres://user:secret@host/db`）。如果
写入密码，请注意书签会原样保存到 `bookmarks.toml`，请妥善处理该文件。

## 布局

```
┌─ Database ──────────────────────────────────────────┐
│ Table ▾ users                                        │
│ id │ name  │ score │ active                          │
│  1 │ alice │ 1.5   │ true                             │
│  2 │ bob   │ 2.25  │ false                            │
│  3 │ carol │ NULL  │ true                             │
└──────────────────────────────────────────────────────┘
```

顶部是**表选择器**，其余部分是**数据表格**。表格有一个二维单元格光标：高亮的单元格
是排序、过滤和复制的目标。共享状态栏显示当前范围、总行数、当前排序和过滤，例如：
`app.db · users · rows 1–200 of 1203 · sort: name ↑ · filter: 1`。

## 按键

| 按键 | 操作 |
|------|------|
| `Tab` | 在表选择器和表格之间切换焦点 |
| `Enter` / `Space`（在选择器上） | 打开表下拉列表 |
| `↑ ↓ ← →` | 移动单元格光标（自动滚动行和列） |
| `PageUp` / `PageDown` | 翻一屏；到边缘时加载下一/上一个窗口 |
| `Home` / `End` | 跳到第一行 / 最后一行 |
| `S` | 按当前列排序——循环：升序 → 降序 → 不排序 |
| `Ctrl+F` / `F3` | 过滤当前列（打开过滤对话框） |
| `Alt+F` | 清除所有过滤 |
| `Space` / `F12`（在表格上） | 显示完整当前行（键/值），并提供复制选项 |
| `Ctrl+C` | 复制当前单元格的值 |
| `Ctrl+Y` | 将当前行复制为制表符分隔的值 |
| `Ctrl+R` | 刷新表列表并重新加载当前视图 |

这些操作键可在配置的 `[database.keybindings]` 中自定义，并显示在帮助面板中。
导航键（Tab、方向键、PageUp/Down、Home/End）是固定的。

**鼠标：** 点击表选择器将其打开，点击列标题切换其排序，点击数据单元格将光标移到该处。

## 排序

在某列上按 `s` 进行服务端排序。循环为**升序 → 降序 → 不排序**；“不排序”不发出
`ORDER BY`，在大表上开销最小。已排序的列在表头显示 `↑`/`↓` 箭头。一次只排序一列。

> 在 PostgreSQL 上，翻阅*未排序*的表时，各页之间的行顺序可能略有不同（引擎的自然
> 顺序在 `OFFSET` 之间不稳定）。如需稳定顺序，请应用排序。

## 过滤

在某列上按 `f` 添加条件。可用的运算符取决于列的类型：

- **文本：** 包含、以…开头、以…结尾、=、≠、为空、不为空
- **数字 / 日期：** =、≠、>、≥、<、≤、为空、不为空
- **布尔：** =、≠、为空、不为空

`包含`/`以…开头`/`以…结尾` 不区分大小写。不同列上的条件以 **AND** 组合；再次过滤
同一列会替换其条件。按 `F` 清除全部。值始终作为绑定参数发送。

## 行详情

在某行上按 `Space` 打开详情对话框，将每一列以键/值列出——适用于长文本、JSON 或
表格放不下的宽行。对话框可将该行复制为 **TSV**、**JSON** 或 **INSERT** 语句。

## 分页

行按窗口获取；内存中只保留当前窗口，因此即使是非常大的表也保持流畅。越过窗口末尾
时会自动加载下一个窗口。总行数（考虑当前过滤）在后台计算，就绪后显示在状态栏中。

## 限制（当前）

- 只读——不支持编辑，不支持任意 SQL。
- 特殊列类型（JSON、数组、UUID、时间戳、无符号整数）在此版本中显示为空/`NULL`；
  常见标量类型完整显示。
- 模式选择（PostgreSQL）默认使用当前模式。
- 暂无应用内密码提示——请使用免密码认证方式或在 URL 中包含密码（见上文）。
