# 操作面板

操作面板展示每一个后台文件系统操作（复制、移动、上传、下载、删除、
批量传输）和每一个后台命令。每个操作都有自己的卡片，包含进度条、
源/目标、已传输字节数和已耗时间。

## 打开与关闭

当第一个后台操作启动时，面板会自动打开。也可以从应用菜单中打开。

关闭遵循标准面板规则（`Alt+X`、`F10`，或在未选中任何操作时按
`Esc`）。当某个操作被高亮时，`Esc` 会取消该操作而不是关闭面板——
详见下文快捷键。

## 快捷键

| 快捷键                          | 操作                                              |
|--------------------------------|---------------------------------------------------|
| `↑` / `↓`（或 `k` / `j`）       | 在操作之间移动光标                                 |
| `Home` / `End`（或 `g` / `G`）  | 跳转到第一个/最后一个操作                          |
| `Space`                        | 暂停或恢复所选操作                                 |
| `Esc` / `Delete` / `Backspace` | 取消所选操作（需确认）                             |

取消操作总是先请求确认，无论由哪个按键（或弹出菜单）触发。
`Esc` 仅在操作被高亮时触发取消；未选中时它沿用应用层默认的
关闭面板行为。

## 单个操作的弹出菜单

点击操作卡片边框中带方括号的类型图标（`[↑]`、`[↓]`、`[⧉]`、`[➜]`
…）会打开一个针对该操作的弹出菜单：

- **Pause / Resume**——与 `Space` 效果相同，切换暂停状态。
- **Cancel**——与 `Esc` / `Delete` 相同；会先请求确认。

操作处于暂停状态时，在图标和标签之间会显示 `⏸` 指示符（例如
`[↑] ⏸ Copy (upload) 23%`）。

## 取消清理

取消已经传输了字节的操作**不会**默默留下垃圾文件。面板会询问如何
处理剩余物：

### 远程上传（单文件）

如果被取消的是远程上传操作，面板会弹出一个模态框：

```
Upload was cancelled

Delete partial upload 'filename'?
```

选择 **Yes**（默认）会向服务器发送删除该传输中文件的命令。**No** 会
将其保留在服务器上——比如稍后通过服务端的断点续传机制继续传输。

### 本地/远程批量操作

对于已经在目标目录中产生文件的批量操作，模态框提供两个选项：

- **Delete copied**——删除批量操作到目前为止写入的所有内容。
- **Keep copied**——保留部分写入的集合不动。

要清理的路径列表来自批量操作跟踪的目标，因此仅影响由本次取消的
操作产生的文件；目标目录中预先存在的文件不会被触及。
