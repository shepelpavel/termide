# 文本编辑器

文本编辑器面板提供功能完善的编辑器，用于处理文本文件，支持多种编程语言的语法高亮。

## 主要功能

- **语法高亮**：自动高亮常用编程语言（Rust、Python、JavaScript、C/C++、Go 等）
- **Git 差异可视化**：与 HEAD 对比的实时变更可视化，行号带背景色高亮（绿色背景表示新增、黄色背景表示修改）并自动选择对比度最佳的前景色，删除标记显示已删除行的数量
- **搜索和替换**：文本搜索，支持区分大小写和替换匹配项
- **编辑历史**：撤销和重做操作
- **剪贴板**：通过系统剪贴板进行复制、剪切和粘贴
- **自动保存**：关闭有未保存更改的文件时提示保存
- **单词导航**：使用 Ctrl+Left/Right 按单词移动光标，使用 Ctrl+Shift+Left/Right 按单词选择
- **自动缩进**：新行自动继承当前行的缩进；在 `{`、`(`、`[`、`:` 之后智能增加一级缩进
- **自动关闭括号**：自动插入匹配的闭合括号和引号（`()`、`[]`、`{}`、`""`、`''`）；输入闭合括号时跳过已有的；在括号对之间按退格键删除两个字符

## 搜索和替换

### 内嵌搜索栏 (Ctrl+F)

按 `Ctrl+F` 打开**停靠在编辑器顶部的搜索栏**（与文件管理器一致），其下有分隔线；
缓冲区仍显示，输入时匹配项即时高亮。`Tab` 在搜索栏与缓冲区之间切换焦点：在缓冲区
区域光标正常移动和滚动，同时搜索栏保持打开；`Tab` 返回搜索栏。

| 快捷键 | 操作 |
|-------------------|--------------------------------------------|
| `Ctrl+F`          | 打开搜索栏                                 |
| 输入文字          | 输入时实时更新搜索                         |
| `F3` / `Enter`    | 转到下一个匹配                             |
| `Shift+F3`        | 转到上一个匹配                             |
| `Tab`             | 在搜索栏与缓冲区之间切换                   |
| 方向键            | 在字段/按钮之间移动（搜索栏内）           |
| `Escape`          | 关闭搜索栏                                 |
| 鼠标点击          | 点击按钮 / 开关                            |

**功能特性：**
- 输入时实时预览；匹配计数（例如"3/12"）
- 导航按钮：◄ 上一个、下一个 ►
- `[.*] Regex` 与 `[Aa] Case` 开关——点击，或将焦点移到按钮行后按 `Enter` /
  `Space`。正则默认关闭（字面搜索）。
- `Ctrl+R` 针对当前缓冲区重新运行搜索（刷新）。
- 关闭搜索栏后保留查询

**搜索栏关闭时的行为：**
- `F3` / `Shift+F3` 在缓冲区中浏览匹配项
- 任何导航/编辑键停用搜索模式
- 使用 `F3` 重新打开时恢复上次的查询

### 内嵌替换栏 (Ctrl+H)

按 `Ctrl+H` 打开带「替换」字段的搜索栏。

| 快捷键 | 操作 |
|-------------------|--------------------------------------------|
| `Ctrl+H`          | 打开搜索/替换栏                            |
| 在查找框中输入    | 输入时实时更新搜索                         |
| `Tab` / 方向键    | 在查找、替换与按钮之间移动                 |
| `F3`              | 转到下一个匹配                             |
| `Shift+F3`        | 转到上一个匹配                             |
| `Enter`（查找框） | 转到下一个匹配                             |
| `Enter`（替换框） | 替换当前匹配                               |
| `Escape`          | 关闭栏                                     |
| 鼠标点击          | 点击按钮（◄ Prev、Next ►、Replace、Replace all）或开关 |

**功能特性：**
- 两个输入框：查找和替换；实时预览 + 匹配计数
- 按钮行顺序：`[Aa]` / `[.*]` 开关、**◄ Prev**、**Next ►**、**Replace**、
  **Replace all**（用 `Tab` / 方向键将焦点移到按钮行，然后按 `Enter` / `Space`，
  或点击）：
  - **◄ Prev** / **Next ►** — 导航匹配项
  - **Replace** — 替换当前匹配并移到下一个
  - **Replace all** — 替换所有匹配项（状态栏报告数量）
- `[.*] Regex` 与 `[Aa] Case` 开关。开启正则时替换字段支持 `$1` / `${name}`
  捕获组；否则按字面插入。
- 关闭栏后保留查找和替换文本。

可配置的编辑器按键 `replace_current` 与 `replace_all` 仍会直接作用于当前搜索
（无论栏是否打开，参见[快捷键](keybindings.md)）。

## 自定义语法高亮

内置语言通过 tree-sitter 高亮。对于尚无 tree-sitter 语法的文件类型——例如你正在
设计的语言——可以在 `config.toml` 中定义**轻量级关键字高亮器**，无需重新编译
TermIDE。它会着色行/块注释、字符串、数字以及你的关键字/类型词表，并复用当前主题
的配色。

```toml
[[highlight.custom_languages]]
name = "Alatyr"
extensions = ["al"]          # 文件扩展名，不带前导点
line_comment = "##"          # 此标记之后到行尾为注释
# block_comment = ["/*", "*/"]  # 可选；仅在单行内匹配
keywords = ["pub", "struct", "enum", "match", "if", "else", "and", "or", "not"]
types = ["u8", "i64", "usize", "ptr"]
```

可为其他文件类型添加更多 `[[highlight.custom_languages]]` 块。对于同一扩展名，
tree-sitter 语法（若存在）始终优先于自定义条目。这是逐行分词器，不跟踪跨行结构
（块注释必须在一行内开始和结束）；当语言稳定后，真正的 tree-sitter 语法可提供更
丰富、带作用域的高亮。

## 鼠标支持

- **单击**：将光标设置到点击位置
- **双击**：选择光标下的单词
- **三击**：选择整行
- **按住并移动**：文本选择
- **Shift+点击** / **Alt+点击**：将当前选区从锚点（若尚无选区，则从光标位置）扩展到点击位置。提供 Alt+点击 是因为 GNOME Terminal / VTE 会吞掉 Shift+点击用于自身的文本选择；在 kitty / alacritty / WezTerm 中两种组合都可用。
- **滚轮**：滚动编辑器内容
- **Ctrl+点击**：跳转到定义（LSP）；或在十六进制颜色值上显示颜色预览（如 `#ff0000`、`#abc`）

**颜色预览：** 按住 Ctrl 并点击十六进制颜色值时，会显示一个包含颜色色块和十六进制代码的小弹窗。弹窗在鼠标按钮按住时保持可见，松开时消失。

**注意：** 鼠标选择在自动换行模式下也能正确工作，会考虑换行的行。

## 自动换行

启用自动换行后（可在设置中配置），长行会自动换行以适应面板宽度。编辑器正确处理：

- **光标定位**：光标导航和显示在换行的行之间正确工作
- **鼠标选择**：即使行跨越多个视觉行，点击和拖动也能准确选择文本
- **行号**：显示逻辑行号，而非视觉行号
- **编辑操作**：所有编辑命令（剪切、复制、粘贴、撤销/重做）与换行内容无缝工作

在配置文件（`~/.config/termide/config.toml`）中启用/禁用自动换行：
```toml
[editor]
word_wrap = true  # 或 false
```

## 自动缩进

启用自动缩进（默认启用）后，按 `Enter` 创建的新行会自动继承当前行的缩进。此外，智能缩进会在以 `{`、`(`、`[` 或 `:` 结尾的行后增加一级缩进。

**示例：**
```
if condition {|        ← 光标在此，按 Enter
    |                  ← 新行，带额外缩进
```

**括号分割缩进：** 在匹配的括号对之间（`{|}`、`(|)`、`[|]`）按 `Enter` 时，编辑器会创建三行——开括号、带缩进的新行（光标位置）和单独一行的闭括号：

```
fn main() {|}          ← 光标在大括号之间，按 Enter
fn main() {
    |                  ← 光标在此，已缩进
}
```

在配置文件中启用/禁用自动缩进：
```toml
[editor]
auto_indent = true  # 或 false（默认：true）
```

## 自动关闭括号

启用自动关闭括号（默认启用）后，输入开括号或引号时自动插入匹配的闭合字符，并将光标置于两者之间。

**支持的括号对：** `()`、`[]`、`{}`、`""`、`''`

**行为：**
- 输入 `(` 时插入 `()`，光标在两者之间
- 当光标处字符已经是 `)` 时，输入 `)` 会跳过而不是插入重复字符
- 在空括号对（如 `()`）之间按 `Backspace` 会删除两个字符
- 在字母数字字符后不会自动关闭引号（以支持如 "it's"、"don't" 中的撇号）

在配置文件中启用/禁用自动关闭括号：
```toml
[editor]
auto_close_brackets = true  # 或 false（默认：true）
```

## 状态栏信息

在编辑器中工作时，状态栏显示：
- 文件名和修改指示器（*）
- 当前光标位置（行:列）
- 制表符宽度（`Tab N`）——**可点击**：打开一个输入模态框，仅为当前编辑器面板覆盖 `tab_size`。该覆盖优先于全局 `[editor].tab_size`，不受逐帧配置重新同步影响，也不会修改 `config.toml`。接受 1..=16，空值或非法输入将被忽略。
- 行尾格式（LF / CRLF）和编码（UTF-8）
- 搜索信息（匹配数量）
- 文件类型（纯文本 / 只读）

## Git 差异可视化

在 Git 仓库中编辑文件且启用了 `show_git_diff` 时，编辑器会显示与 HEAD 对比的实时差异信息：

### 行号颜色

行号使用背景色高亮来显示与 HEAD 相比的状态。文字颜色自动选择以获得最佳对比度：

- **绿色背景** - 行已添加（不在 HEAD 中）
- **黄色背景** - 行已修改（与 HEAD 不同）
- **红色标记 (▶)** - 标记删除点（此行之后有行被删除）
- **默认颜色** - 行与 HEAD 相同未变

### 删除标记

当有行被删除时，会插入一个虚拟行来可视化删除：

- 显示横线（`━`）横跨编辑器宽度
- 在行号区域显示红色的删除标记字符（`▶`）
- 显示居中文本："已删除 N 行"（例如"已删除 3 行"）
- 使用灰色/禁用颜色样式以区别于实际内容
- 不影响行号编排（显示 `▶` 而非数字）

**示例：**
```
  42 | function calculateTotal() {
 ▶   | ━━━━━━━ 已删除 5 行 ━━━━━━━
  43 |     return result;
```

### 工作原理

- **自动更新**：保存文件时更新差异
- **实时对比**：将当前缓冲区内容与 HEAD 版本对比
- **撤销/重做支持**：撤销/重做删除操作时标记会出现/消失
- **与编辑兼容**：所有正常编辑操作与差异可视化无缝协作

### 配置

在配置文件（`~/.config/termide/config.toml`）中启用或禁用 Git 差异可视化：

```toml
# 在行号上显示 git diff 颜色（默认：true）
show_git_diff = true
```

**注意：**
- 仅在 Git 仓库中编辑文件时有效
- 需要文件存在于 HEAD 中（新的未跟踪文件会将所有行显示为已添加）
- 虚拟删除标记行仅用于可视化，不影响文件内容

## Git Blame 注解

在 git 仓库中编辑文件时，编辑器会在光标所在行末尾显示内联 blame 注解，显示最后修改该行的作者、时间、提交哈希和提交描述。

### 内联注解

注解以暗色显示在当前行末尾：

```
  some_function();                          nvn, 2 weeks ago • abc1234 Fix bug
```

### 控制

通过设置模态框（Alt+P → Editor → Show Blame）切换 blame 注解的开关。

**注意：** 在 git 仓库中打开文件时，blame **默认启用**。注解在后台异步加载，`git blame` 进程完成后显示。

### 配置

```toml
[editor]
show_blame = true  # 或 false（默认：true）
```

## LSP（语言服务器协议）

TermIDE 内置 LSP 支持，提供智能代码辅助。当配置了可用的语言服务器时，您将获得：

- **代码补全** - 输入时的上下文感知建议
- **诊断** - 实时错误和警告指示
- **加载状态** - 面板标题中的转圈动画显示服务器状态（启动中/索引中）

### 触发补全

| 快捷键 | 操作 |
|-------------------|--------------------------------------------|
| `Ctrl+J` / `Ctrl+Space` | 手动触发补全弹出框                   |
| `Enter`           | 接受选中的补全                             |
| `Escape`          | 关闭补全弹出框                             |
| `↑` / `↓`         | 浏览建议列表                               |
| 输入字符          | 通过输入过滤建议                           |

**自动补全：** 启用时（默认），补全弹出框自动出现：
- 输入标识符字符（字母、数字、`_`）后
- 输入触发字符（`.`、`:`、`(`、`<`）后立即出现

### 补全弹出框

补全弹出框显示：
- **图标** - 表示项目类型（函数 ƒ、变量 v、类 C 等）
- **标签** - 补全文本
- **滚动指示器** - 列表可滚动时显示 ▲/▼

### 配置

在配置文件（`~/.config/termide/config.toml`）中配置 LSP：

```toml
[lsp]
enabled = true              # 启用/禁用 LSP（默认：true）
auto_completion = true      # 输入时自动触发补全（默认：true）
completion_delay_ms = 100   # 自动补全延迟（默认：100）

[lsp.servers.rust]
command = "rust-analyzer"
args = []
root_markers = ["Cargo.toml"]

[lsp.servers.python]
command = "pylsp"
args = []
root_markers = ["pyproject.toml", "setup.py", "requirements.txt"]

[lsp.servers.typescript]
command = "typescript-language-server"
args = ["--stdio"]
root_markers = ["package.json", "tsconfig.json"]
```

### 查找引用

| 快捷键              | 操作                                       |
|-------------------|--------------------------------------------|
| `Shift+F12`       | 查找光标下符号的所有引用                    |

打开引用面板，列出符号被使用的所有位置。点击任何引用即可导航到该位置。

### 重命名符号

| 快捷键              | 操作                                       |
|-------------------|--------------------------------------------|
| `F4`              | 重命名光标下的符号                          |

打开输入对话框输入新名称。项目中的所有出现位置将通过 LSP WorkspaceEdit 更新。

### 代码操作（Code Actions）

| 快捷键              | 操作                                       |
|-------------------|--------------------------------------------|
| `Alt+Enter`       | 显示光标处的代码操作 / 快速修复             |

请求当前行的快速修复（例如 PHP 中用于添加 `use` 语句的「Import class」），并在弹出列表中显示。使用 `↑`/`↓` 选择，`Enter` 应用，`Esc` 关闭。所选操作的编辑将跨文件应用，并重新加载已打开的编辑器。（`Ctrl+.` 已绑定为切换注释，因此默认使用 `Alt+Enter`。）

### 服务器状态指示器

打开支持 LSP 的文件时，面板标题显示加载动画：
- `⠋ main.rs (starting)` — 服务器正在启动
- `⠋ main.rs (indexing)` — 服务器正在索引项目
- `main.rs` — 服务器就绪

### 支持的语言

LSP 可与任何实现了 LSP 协议的语言服务器配合使用。常见示例：
- **Rust** - rust-analyzer
- **Python** - pylsp、pyright
- **TypeScript/JavaScript** - typescript-language-server
- **Go** - gopls
- **C/C++** - clangd

**注意：** 您需要单独安装语言服务器。TermIDE 仅提供 LSP 客户端集成。

## Vim 模式

TermIDE 包含可选的 Vim 风格编辑模式，完整支持西里尔文键盘。

### 启用 Vim 模式

在配置文件（`~/.config/termide/config.toml`）中启用 Vim 模式：

```toml
vim_mode = true
```

### 可用模式

| 模式 | 描述 | 指示器 |
|------|-------------|-----------|
| 普通模式 | 导航和命令 | （无） |
| 插入模式 | 文本输入 | `-- INSERT --` |
| 可视模式 | 字符选择 | `-- VISUAL --` |
| 可视行模式 | 行选择 | `-- VISUAL LINE --` |

### 模式切换

| 按键 | 操作 |

| `Escape` | 返回普通模式 |
| `i` | 在光标前插入 |
| `I` | 在行首插入 |
| `a` | 在光标后追加 |
| `A` | 在行尾追加 |
| `o` | 在下方插入新行 |
| `O` | 在上方插入新行 |
| `v` | 进入可视模式 |
| `V` | 进入可视行模式 |

### 移动键（普通/可视模式）

| 按键 | 操作 |
|-----|--------|
| `h` / `←` | 向左移动 |
| `j` / `↓` | 向下移动 |
| `k` / `↑` | 向上移动 |
| `l` / `→` | 向右移动 |

### 西里尔文键盘支持

Vim 模式与西里尔文键盘布局无缝配合。使用俄语或其他西里尔文布局输入时，所有 Vim 命令会自动翻译：

- `о`（俄语）→ `j`（向下移动）
- `л`（俄语）→ `k`（向上移动）
- `н`（俄语）→ `y`（复制）
- `в`（俄语）→ `d`（删除）

这使您无需切换键盘布局即可使用 Vim 命令。

### 可视模式操作

在可视或可视行模式下：
- 使用移动键扩展选区
- `d` 或 `x` - 删除选区
- `y` - 复制选区
- `c` - 修改选区（删除并进入插入模式）
- `>` - 增加缩进
- `<` - 减少缩进
