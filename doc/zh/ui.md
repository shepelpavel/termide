# 应用程序窗口概览

**应用程序占据所有可用空间，垂直划分为以下元素：**
- 菜单栏
- 面板区域
- 状态栏

应用程序还使用弹出窗口：
- 帮助
- 设置
- 关闭应用确认

## 模态窗口

应用程序使用交互式模态窗口进行各种操作：
- **内嵌查找 / 替换栏**（`Ctrl+F` / `Ctrl+H`）- 停靠在编辑器、文件管理器和终端内的搜索与替换，带实时预览、匹配计数、`[Aa]` / `[.*]` 开关和导航按钮
- **输入模态框** - 用于文件操作（创建、重命名等）的各种提示
- **确认对话框** - 删除确认、未保存更改等

**模态框功能：**
- 模态框标题栏中的 `[X]` 关闭按钮（可用鼠标点击）
- 使用 Tab/Shift+Tab 进行键盘导航（在模态框字段/按钮之间）
- 所有按钮支持鼠标操作
- Escape 键关闭模态框
- 搜索/替换操作的实时预览
- 状态保持（保存上次输入的文本）

### 设置模态框（Settings）

通过 `Alt+P` 或菜单 **Options → Edit preferences** 打开。与之前的横向标签布局不同，现已改为 **侧边栏布局**：

**三个区域：**
- **左侧边栏**（宽 18 列）— 区块列表，使用 Up/Down 浏览
- **内容区**— 所选区块的字段，按 `── 分组 ──` 副标题归类并用空行分隔
- **底部按钮**— `Apply & Save`、`Reset`、`Cancel`

**侧边栏区块：**
- General、Editor、File Manager、Terminal、LSP、Logging、VFS — 普通叶节点
- **Keybindings** — 可展开分组（▶/▼），包含 7 个子区块（Global、Editor、FileManager、GitStatus、GitDiff、GitLog、Terminal）

**导航：**

| 快捷键 | 操作 |
|--------|------|
| `Up` / `Down` | 在当前区域内移动（区块 / 字段 / 按钮） |
| `Tab` / `Shift+Tab` | 在区域之间循环切换焦点（Sidebar ↔ Content ↔ Buttons） |
| `Left` / `Right` | **不** 切换区域；用于修改值（枚举循环、布尔切换），以及在侧边栏中折叠 Keybindings 分组 |
| `Enter` / `Space` | 激活：在侧边栏中进入区块；在内容区中切换 bool/enum 或开始编辑数值/文本；在分组头上切换展开 |
| `Escape` | 关闭模态框（Cancel） |
| 鼠标滚轮 | 滚动侧边栏 / 内容 |

**字段与指示符：**
- **Bool** — `[✓]`（开）或 `[✗]`（关），通过 `Enter`/`Space` 或 `Left`/`Right` 切换
- **Enum** — `< 值 >`，通过 `Left`/`Right` 循环
- **Number** / **OptionalText** — 按 `Enter` 进入内联编辑模式
- **LSP → Servers** — 服务器列表项以项目符号 `•` 开头；另外还有一行 `+ Add Server`

**Keybindings — 按键捕获：**  
在绑定列表中用 `Up/Down` 导航，按 `Enter` 进入 Capturing 模式（下一次按键将成为新绑定），`Delete`/`Backspace` 清除绑定。

当焦点离开侧边栏时，当前区块的高亮会取消——因为当前区块名已显示在内容区标题中，再双重高亮会让用户分不清真正的焦点位置。

## 菜单栏

菜单栏位于窗口顶部，包括：左侧的菜单项、系统资源指示器（CPU、内存、网络速度），以及右侧的 HH:MM 格式时钟。
菜单的激活/停用及各项目可通过鼠标点击或[键盘快捷键](#键盘导航和面板管理)访问。

**菜单项：**
- `Sessions` — 会话管理子菜单：
  - 新建会话 — 在新目录中创建会话
  - 切换会话 — 打开会话切换模态窗口
  - 更改根路径 — 将当前会话移至其他目录
- `Windows` — 面板创建子菜单：
  - Files — 文件管理器面板
  - Terminal — 终端面板（有子菜单可选择 shell：列出系统中所有可用的 shell，默认 shell 以 ● 标记）
  - Editor — 文本编辑器面板
  - Git Status — Git 状态面板
  - Git Log — 提交历史面板
  - Git Stash — Git 暂存管理面板
  - Journal — 应用日志面板
  - Diagnostics — LSP 诊断面板
  - [Operations](operations.md) — 后台操作面板
  - Outline — 代码结构导航面板
- `Scripts` — 用户自定义脚本（支持分组子菜单）。点击分组标题可展开子菜单；再次点击同一标题则折叠（toggle）。
- `Bookmarks` — 书签（目录、文件、SSH、SFTP）。分组行为与 Scripts 相同，支持 toggle。
- `Options` — 设置子菜单：
  - Themes — 主题选择（实时预览）
  - Language — 界面语言（实时预览）
  - Manage scripts — 打开脚本文件夹
  - Manage bookmarks — 打开书签文件
  - Edit preferences — 在编辑器中打开 config.toml
  - Help — 帮助面板
  - Quit — 退出应用

**系统资源指示器**（均可点击——打开详情模态框）：

| 指示器 | 说明 | 点击打开 |
|--------|------|---------|
| `CPU XX%` | CPU 使用率 | CPU 占用前 10 的进程 |
| `RAM X/YGB` | 内存使用 | 内存占用前 10 的进程 |
| `↓…/↑…` | 网络速度（↓ 下载，↑ 上传） | 网络活动前 10 的进程 |
| `设备 已用/总计GB`（状态栏） | 磁盘使用 | 磁盘占用前 10 的进程 + 文件系统分区 |

再次点击同一指示器会关闭其打开的窗口（toggle）；状态栏的磁盘指示器行为相同。

颜色编码：绿色 < 50%，黄色 50–75%，红色 > 75%。

## 面板区域

面板区域填充菜单栏和状态栏之间的垂直空间，从左到右延伸至窗口边缘。
布局会根据终端宽度自适应，在更宽的屏幕上显示更多面板组。面板（终端、编辑器、Git）可以打开并排列在旁边。

**可打开的面板类型：**
- [文件管理器](file-manager.md) — `Alt+F`
- [终端](terminal.md) — `Alt+T`
- [文本编辑器](editor.md) — `Alt+E`
- Git 状态 — `Alt+G`
- 大纲 — `Alt+O`
- 诊断 — `Alt+I`
- Git 日志 — `Alt+C`
- Git 差异
- [操作](operations.md)
- 图片查看器
- 帮助 — `Alt+H`
- 日志 — `Alt+L`

**可关闭面板的特点：**
- 面板标题中有 `[≡]` 操作按钮（点击打开上下文菜单：关闭 / 拆分 / 合并 / 移动）
- 可通过 Escape、Alt+X 或 F10 关闭
- 列宽通过 `Alt+=` / `Alt+-` 调整
- 列内每个面板的高度通过 `Alt+Shift+=` / `Alt+Shift+-`（步长 1 行）、用鼠标拖动面板的下边框或在列内上下拖动面板标题进行调整
- `Alt+F11` 切换聚焦面板的全屏预设（一个面板占满整列，其余各自缩为一行标题）；再次按下恢复之前的高度
- 可通过拖拽顶部边框来移动（参见下面的"鼠标交互"）

**面板操作上下文菜单：**

点击面板标题中的 `[≡]` 按钮（或在活动面板上按 `Alt+K` / `Shift+F10`）会打开下拉菜单：
- **上移 / 下移** — 在当前组内重新排序（当组内有多个面板时）
- **左移 / 右移** — 移动到相邻组（当有多个组时）
- **拆分 / 合并** — 从组中拆分出新组（当堆叠时），或将单独面板与相邻组合并
- **关闭** — 关闭面板（必要时带确认）

菜单项根据上下文过滤：例如，只有一个面板在一个组中时，*拆分/合并* 隐藏；单独面板时，*上移/下移* 隐藏。

**鼠标交互：**
- 点击标题区域激活面板；双击文件管理器标题打开目录选择器。
- **拖拽面板的底部边框** 在列内调整该面板的高度（实时 `━` 幽灵线
  预览新分隔线的位置，调整在释放时应用）。
- **拖拽面板的顶部边框（标题）** —— resize 或 reorder：
  - 在源面板主体或上方相邻面板的主体内释放 → 垂直 resize：
    被拖面板上方的分隔线跟随光标（列顶部的面板上方没有分隔线，
    会回退到移动模式）。
  - 在同组另一个面板的标题行上释放 → 重新排序：根据方向插入
    到目标的前面 / 后面。
  - 在另一列面板的主体上释放 → 移动到该组，并在 drop 行处分割
    目标面板。
  - 在两个组之间的 2 格间隙释放 → 在该位置创建新组。
  - 在最右组之外释放 → 作为新的最后一组追加。
  - 拖拽过程中，粗 `━` 线标记水平 drop 区（列内 resize / split），
    粗 `┃` 线标记垂直 drop 区（组之间的新列）。
  - 拖拽过程中按 `Escape` 取消。

## 状态栏

状态栏用于显示活动面板的附加工作信息。
根据活动面板的类型，显示相应的数据。

### 磁盘空间指示器

状态栏在右侧显示磁盘空间信息，格式为：`设备名 已用/总计GB (使用率%)`，根据使用水平进行颜色编码：

**颜色编码：**
- **绿色** 磁盘使用率 < 50%
- **黄色** 磁盘使用率 50-75%
- **红色** 磁盘使用率 > 75%

**格式：** `设备名 已用/总计 (使用率%)`

示例：`NVME0N1P2 386/467Gb (83%)`

设备名称从文件系统自动检测：
- Linux 上：显示分区名称，如 `NVME0N1P2`、`SDA1` 等
- macOS 上：显示磁盘标识符
- 显示的设备对应当前目录所在的分区

### Caps Lock

Termide 通过 `KeyboardEnhancementFlags::DISAMBIGUATE_ESCAPE_CODES | REPORT_ALTERNATE_KEYS | REPORT_EVENT_TYPES` 启用 [Kitty 键盘协议](https://sw.kovidgoyal.net/kitty/keyboard-protocol/)。`REPORT_EVENT_TYPES` 会在每次按键事件中附带 `KeyEventState::CAPS_LOCK`，热键匹配器据此忽略 X11/Linux 终端在 Caps Lock 开启时强行附加的虚假 `Shift` 修饰符。

实际效果：

- 开启 Caps Lock 时 `Alt+T` 这类绑定依然生效（否则事件到达时是 `{Char('T'), Alt|Shift}`，无法匹配 `{Char('t'), Alt}`）。
- 真正的 `Shift+字母` 绑定（例如 `Ctrl+Shift+F` 用于内容搜索）仍与无 Shift 版本区分清楚——只在真的报告了 Caps Lock 时才剥离 Shift。
- 不支持 Kitty 协议的终端（不发送 Caps Lock 位）会回退到严格的修饰符比较，行为保持不变——在这些终端中建议关闭 Caps Lock 使用。

内嵌终端模拟器的方向键和 Home/End 的修饰编码见 [terminal.md](terminal.md#交互)。
